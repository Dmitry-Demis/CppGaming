Преобразование типов и static_cast

В C++ значения разных типов часто нужно комбинировать: складывать int с double, присваивать long в short, передавать float в функцию, ожидающую int. Компилятор автоматически выполняет преобразования типов, но не все они безопасны. В этом разделе мы разберём, как работают неявные и явные преобразования, и когда использовать static_cast.

Неявное преобразование типов

Неявное преобразование типов (implicit type conversion) — автоматическое преобразование значения из одного типа в другой, выполняемое компилятором без явного указания программиста. Также называется автоматическим преобразованием или приведением типов (type coercion).

Расширяющие преобразования (Widening conversions)

Расширяющие преобразования — преобразования, при которых значение меньшего типа помещается в больший тип без потери данных:

C++ 
#include <iostream>

int main() {
    int i = 42;
    double d = i;  // int > double (расширяющее преобразование)
    
    std::cout << "i = " << i << '\n';  // 42
    std::cout << "d = " << d << '\n';  // 42.0
    
    // Безопасные расширяющие преобразования:
    short s = 100;
    int i2 = s;        // short > int ?
    long l = i2;       // int > long ?
    long long ll = l;  // long > long long ?
    
    float f = 3.14f;
    double d2 = f;     // float > double ?
    
    return 0;
}
Безопасные расширяющие преобразования

Расширяющие преобразования между целочисленными типами ($\text{short} \to \text{int} \to \text{long} \to \text{long long}$) и от float к double всегда безопасны — значение сохраняется точно.

Сужающие преобразования (Narrowing conversions)

Сужающие преобразования — преобразования, при которых значение большего типа помещается в меньший тип. Это может привести к потере данных:

C++ 
#include <iostream>

int main() {
    double d = 3.14159;
    int i = d;  // double > int (сужающее преобразование)
    
    std::cout << "d = " << d << '\n';  // 3.14159
    std::cout << "i = " << i << '\n';  // 3 (дробная часть отброшена!)
    
    // Опасные сужающие преобразования:
    int big = 100000;
    short s = big;  // int > short (переполнение!)
    
    std::cout << "big = " << big << '\n';  // 100000
    std::cout << "s = "   << s   << '\n';  // -31072 (неопределённое поведение!)
    
    return 0;
}
? Опасность сужающих преобразований

Сужающие преобразования могут привести к:

  • Потере точности: $3.14159 \to 3$
  • Переполнению: $100\,000$ не помещается в short (макс. $32\,767$)
  • Неопределённому поведению: результат непредсказуем

Явное преобразование типов

Чтобы сделать преобразование типов явным и показать, что вы осознаёте возможную потерю данных, используйте явное приведение типов (explicit type casting).

C-style cast (устаревший)

C++ — C-style cast 
double d = 3.14;
int i = (int)d;  // C-style cast (не рекомендуется в C++)
? Не используйте C-style cast в C++

C-style cast (int)value слишком мощный и может выполнять опасные преобразования (например, reinterpret_cast). В C++ используйте static_cast.

static_cast — безопасное явное преобразование

static_cast — оператор явного преобразования типов в C++, выполняющий проверки на этапе компиляции. Используется для безопасных преобразований между совместимыми типами.

Синтаксис:

C++ 
static_cast<новый_тип>(выражение)

Примеры:

C++ 
#include <iostream>

int main() {
    // 1. Явное преобразование double > int
    double d = 3.14159;
    int i = static_cast<int>(d);
    std::cout << "d = " << d << ", i = " << i << '\n';  // d = 3.14159, i = 3
    
    // 2. Явное преобразование int > double для точного деления
    int a = 7;
    int b = 2;
    
    double result1 = a / b;  // 3 (целочисленное деление!)
    double result2 = static_cast<double>(a) / b;  // 3.5 (правильно!)
    
    std::cout << "7 / 2 = " << result1 << '\n';  // 3
    std::cout << "7.0 / 2 = " << result2 << '\n';  // 3.5
    
    // 3. Явное преобразование char > int
    char ch = 'A';
    int ascii = static_cast<int>(ch);
    std::cout << "Символ '" << ch << "' имеет код " << ascii << '\n';  // 65
    
    return 0;
}

Числовые повышения (Numeric promotions)

При выполнении арифметических операций компилятор автоматически повышает типы операндов до более широкого типа:

C++ 
#include <iostream>
#include <typeinfo>

int main() {
    short s1 = 10;
    short s2 = 20;
    
    // short + short > int + int > int
    auto result = s1 + s2;
    
    std::cout << "Тип result: " << typeid(result).name() << '\n';  // int
    std::cout << "result = " << result << '\n';  // 30
    
    // Смешанные типы: int + double > double + double > double
    int i = 5;
    double d = 2.5;
    auto result2 = i + d;
    
    std::cout << "Тип result2: " << typeid(result2).name() << '\n';  // double
    std::cout << "result2 = " << result2 << '\n';  // 7.5
    
    return 0;
}
Правила числовых повышений

При арифметических операциях компилятор повышает типы в следующем порядке:

$\text{char/short} \to \text{int} \to \text{long} \to \text{long long} \to \text{float} \to \text{double} \to \text{long double}$

Результат операции имеет тип наиболее широкого операнда.

Brace-initialization и защита от сужения

C++11 ввёл uniform initialization (инициализацию фигурными скобками), которая запрещает сужающие преобразования на этапе компиляции:

C++11 
#include <iostream>

int main() {
    // Старый стиль — разрешает сужающие преобразования
    int i1 = 3.14;  // ? компилируется, но теряется точность
    std::cout << "i1 = " << i1 << '\n';  // 3
    
    // Brace-init — запрещает сужающие преобразования
    // int i2{3.14};  // ? ошибка компиляции: narrowing conversion
    
    // Явное преобразование с brace-init
    int i3{static_cast<int>(3.14)};  // ? OK, явно указано
    std::cout << "i3 = " << i3 << '\n';  // 3
    
    // Переполнение тоже запрещено
    // short s{100000};  // ? ошибка: 100000 не помещается в short
    
    return 0;
}
Рекомендация: используйте brace-init

Всегда используйте {} для инициализации переменных — это защитит вас от случайных сужающих преобразований и потери данных.

Частые ошибки

Целочисленное деление

C++ 
#include <iostream>

int main() {
    int a = 7;
    int b = 2;
    
    // ? НЕПРАВИЛЬНО: int / int = int
    double result1 = a / b;
    std::cout << "7 / 2 = " << result1 << '\n';  // 3.0 (не 3.5!)
    
    // ? ПРАВИЛЬНО: преобразуем хотя бы один операнд в double
    double result2 = static_cast<double>(a) / b;
    std::cout << "7.0 / 2 = " << result2 << '\n';  // 3.5
    
    // Альтернатива: умножение на 1.0
    double result3 = 1.0 * a / b;
    std::cout << "1.0 * 7 / 2 = " << result3 << '\n';  // 3.5
    
    return 0;
}

Переполнение при преобразовании

C++ 
#include <iostream>
#include <limits>

int main() {
    int big = 100000;
    
    // ? Переполнение: 100000 не помещается в short (макс. 32767)
    short s = static_cast<short>(big);
    std::cout << "big = " << big << '\n';  // 100000
    std::cout << "s = " << s << '\n';      // -31072 (неопределённое поведение!)
    
    // ? Проверка перед преобразованием
    if (big <= std::numeric_limits<short>::max() && 
        big >= std::numeric_limits<short>::min()) {
        short safe = static_cast<short>(big);
        std::cout << "Безопасное преобразование: " << safe << '\n';
    } else {
        std::cout << "Ошибка: значение не помещается в short!\n";
    }
    
    return 0;
}

Заключение

Ключевые моменты
  • Неявные преобразования выполняются автоматически, но могут быть опасны
  • Расширяющие преобразования безопасны (short > int > long)
  • Сужающие преобразования могут привести к потере данных
  • Используйте static_cast<T>() для явных преобразований
  • Избегайте C-style cast (int)value в C++
  • Используйте {} для инициализации — защита от сужения
  • Проверяйте диапазоны перед преобразованием в меньший тип
ISO/IEC 14882 — [expr.static.cast] §7.6.1.9 «The result of the expression static_cast<T>(v) is the result of converting the value of v to type T. If T is a reference type, the result is an lvalue; otherwise, the result is a prvalue.»
