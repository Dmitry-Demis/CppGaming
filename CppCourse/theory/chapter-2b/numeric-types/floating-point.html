Типы с плавающей точкой

Типы с плавающей точкой предназначены для вещественных чисел: дробных величин, физических параметров, координат. В отличие от целых чисел они дают приближённое представление — и понимание этой особенности критически важно для корректного кода.

Стандарт IEEE 754

Большинство платформ реализуют типы с плавающей точкой согласно стандарту IEEE 754-2008. Число хранится в виде:

$$\text{значение} = (-1)^s \times m \times 2^e$$

Где $s$ — знак (1 бит), $m$ — мантисса, $e$ — экспонента. Фиксированное число битов определяет точность и диапазон каждого типа.

float IEEE 754 binary32 4 байта ~7 3.14f
double IEEE 754 binary64 8 байт ~15–16 3.14 (без суффикса)
long double платформенный 10–16 байт ~18–19 3.14L
[basic.fundamental] §6.7.1 — ISO/IEC 14882 There are three floating-point types: float, double, and long double. The type double provides at least as much precision as float, and the type long double provides at least as much precision as double.
?? double — тип по умолчанию

Литерал 3.14 без суффикса имеет тип double. Суффикс f делает его float, суффикс L — long double. На практике double является основным выбором.

Точность и ошибки округления

Большинство десятичных дробей не могут быть точно представлены в двоичной системе. Число 0.1 в двоичном — бесконечная дробь; компьютер округляет её до ближайшего представимого значения:

C++ 
#include <iostream>
#include <iomanip>

int main() {
    float  f{0.1f + 0.2f};
    double d{0.1  + 0.2};

    std::cout << std::setprecision(20);
    std::cout << "float  0.1+0.2 = " << f << '\n';
    std::cout << "double 0.1+0.2 = " << d << '\n';
    std::cout << "double 0.3     = " << 0.3 << '\n';

    // Прямое сравнение ненадёжно!
    std::cout << std::boolalpha;
    std::cout << "(d == 0.3): " << (d == 0.3) << '\n'; // false!
    return 0;
}
? Никогда не используйте == для сравнения вещественных чисел

Из-за ошибок округления a == b может вернуть false даже для математически равных значений. Используйте сравнение через эпсилон:

C++ 
#include <cmath>

// Абсолютное сравнение — для чисел близких к нулю
bool nearlyEqual(double a, double b, double eps = 1e-9) {
    return std::abs(a - b) < eps;
}

// Относительное — для больших чисел
bool relativelyEqual(double a, double b, double relTol = 1e-9) {
    return std::abs(a - b) <= relTol * std::max(std::abs(a), std::abs(b));
}

int main() {
    double a{0.1 + 0.2};
    double b{0.3};

    // ? Не делайте так:
    // if (a == b) ...

    // ? Правильно:
    if (nearlyEqual(a, b)) { /* равны с точностью до 1e-9 */ }
}

Машинный эпсилон и numeric_limits

Машинный эпсилон — наименьшее $\varepsilon > 0$ такое, что $1.0 + \varepsilon \neq 1.0$. Характеризует относительную точность типа.

float epsilon ~1.19 ? 10??
double epsilon ~2.22 ? 10???
float digits10 7
double digits10 15
C++ 
#include <iostream>
#include <limits>

int main() {
    std::cout << "float  eps     = " << std::numeric_limits<float>::epsilon()   << '\n';
    std::cout << "double eps     = " << std::numeric_limits<double>::epsilon()  << '\n';
    std::cout << "float  max     = " << std::numeric_limits<float>::max()       << '\n';
    std::cout << "double max     = " << std::numeric_limits<double>::max()      << '\n';
    std::cout << "float  digits  = " << std::numeric_limits<float>::digits10    << '\n';
    std::cout << "double digits  = " << std::numeric_limits<double>::digits10   << '\n';
    return 0;
}

Специальные значения IEEE 754

ЗначениеКак получитьПроверка
+Infinity1.0 / 0.0std::isinf(x) && x>0
-Infinity-1.0 / 0.0std::isinf(x) && x<0
NaN0.0/0.0, sqrt(-1)std::isnan(x) — NaN?NaN!
-0.0underflow с отриц. знакомstd::signbit(x)
C++ 
#include <iostream>
#include <cmath>
#include <limits>

int main() {
    double inf{std::numeric_limits<double>::infinity()};
    double nan{std::numeric_limits<double>::quiet_NaN()};

    std::cout << std::boolalpha;
    std::cout << "1/0       = " << (1.0/0.0)     << '\n'; // inf
    std::cout << "inf+1     = " << (inf + 1)      << '\n'; // inf
    std::cout << "inf-inf   = " << (inf - inf)     << '\n'; // nan

    // NaN не равен самому себе — уникальное свойство IEEE 754
    std::cout << "nan==nan  = " << (nan == nan)    << '\n'; // false!
    std::cout << "isnan(nan)= " << std::isnan(nan) << '\n'; // true
    std::cout << "isinf(inf)= " << std::isinf(inf) << '\n'; // true
    return 0;
}
Когда использовать какой тип

• double — основной выбор; достаточная точность для большинства задач.
• float — когда важны память или скорость (GPU, большие массивы).
• long double — высокоточные математические вычисления.
