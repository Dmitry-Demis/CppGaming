    Псевдонимы и вывод типов

    Псевдонимы типов

    Псевдоним типа (type alias) — это новое имя для существующего типа. Компилятор просто подставляет исходный тип везде, где встречает псевдоним. Псевдоним не создаёт новый тип — это просто другое имя для того же самого.

    Синтаксис: using

    C++ — объявление псевдонима 
    #include <iostream>

int main()
{
    using Distance = double; // Distance — псевдоним для double

    Distance milesToDestination { 3.4 }; // на самом деле double
    std::cout << milesToDestination << '\n'; // 3.4

    return 0;
}

    typedef — устаревший синтаксис

    C++ — typedef vs using 
    // Старый стиль (C и ранний C++)
typedef double Distance;

// Современный стиль (C++11+) — предпочтительно
using Distance = double;

// Для сложных типов using читается лучше:
typedef int (*FcnType)(double, char); // имя FcnType спрятано в середине
using FcnType = int(*)(double, char); // имя FcnType сразу слева

    Псевдоним — не новый тип

    Компилятор не различает псевдонимы одного типа. Это может привести к семантическим ошибкам:

    C++ — псевдонимы взаимозаменяемы 
    using Miles = long;
using Speed = long;

Miles distance { 5 };
Speed mhz { 3200 };

distance = mhz; // компилятор не возражает: оба long
                // но семантически это бессмысленно!

    Когда псевдонимы полезны

    C++ — практические применения 
    #include <string>
#include <vector>
#include <utility>

// 1. Упрощение сложных типов
using VectPairSI = std::vector<std::pair<std::string, int>>;

bool hasDuplicates(VectPairSI pairlist) { return false; }

// 2. Документирование смысла возвращаемого значения
using TestScore = int;
TestScore gradeTest(); // сразу понятно, что возвращается

// 3. Кроссплатформенные типы (как в стандартной библиотеке)
// std::int32_t — это просто псевдоним для int или long
// в зависимости от платформы

// 4. Упрощение обслуживания кода
using StudentId = short; // позже можно изменить на long в одном месте
    💡 Именование

    Называйте псевдонимы с заглавной буквы, без суффиксов: Distance, TestScore, VectPairSI. Это отличает их от переменных и функций.

    Лучшая практика

    Предпочитайте using вместо typedef. Используйте псевдонимы там, где они реально улучшают читаемость или упрощают обслуживание кода.
