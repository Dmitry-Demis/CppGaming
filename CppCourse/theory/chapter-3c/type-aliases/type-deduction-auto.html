    Вывод типа: ключевое слово auto

    В C++ есть небольшая избыточность: когда тип переменной и тип инициализатора совпадают, мы пишем одно и то же дважды. Вывод типа (type deduction) позволяет компилятору самому определить тип из инициализатора.

    C++ — auto выводит тип из инициализатора 
    int main()
{
    auto d { 5.0 };    // double (5.0 — double-литерал)
    auto i { 1 + 2 };  // int (1+2 → int)
    auto x { i };      // int (i — int)

    // Суффиксы работают с auto
    auto a { 1.23f };  // float
    auto b { 5u };     // unsigned int

    // const и constexpr тоже работают
    const auto c { 5 };     // const int
    constexpr auto e { 5 }; // constexpr int (const int)

    return 0;
}

    auto отбрасывает const

    C++ — const теряется при выводе 
    const int a { 5 };  // const int
auto b { a };       // int (const отброшен!)
const auto c { a }; // const int (const добавлен явно)

    Ловушка со строками

    C++ — строковые литералы и auto 
    #include <string>
#include <string_view>

auto s1 { "Hello" };    // const char*, НЕ std::string!

// Если нужна std::string или std::string_view:
using namespace std::literals;
auto s2 { "Hello"s };   // std::string
auto s3 { "Hello"sv };  // std::string_view

    Плюсы и минусы auto

    ✅ Плюсы
    • Имена переменных выровнены — код читается лучше
    • Нельзя забыть инициализировать (auto x; — ошибка компиляции)
    • Нет неожиданных преобразований при инициализации
    • Для сложных типов экономит много символов
    ⚠️ Минусы
    • Тип не виден в коде — нужен IDE для подсказки
    • Легко ошибиться с типом литерала: auto y { 5 } — int, а не double
    • При изменении типа инициализатора тип переменной тоже меняется
    C++ — типичная ошибка с auto 
    #include <iostream>

int main()
{
    auto x { 3 };
    auto y { 2 };
    std::cout << x / y << '\n'; // 1, а не 1.5!
    // Оба int → целочисленное деление

    // Правильно, если нужно вещественное:
    auto a { 3.0 };
    auto b { 2.0 };
    std::cout << a / b << '\n'; // 1.5

    return 0;
}
    Лучшая практика

    Используйте auto, когда тип очевиден из инициализатора или не важен. Указывайте тип явно, когда нужен конкретный тип или когда это улучшает читаемость.
