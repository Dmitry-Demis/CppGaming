    Псевдонимы и вывод типов

    Вывод типа возвращаемого значения функции

    С C++14 ключевое слово auto можно использовать вместо типа возвращаемого значения функции. Компилятор выведет тип из оператора return.

    Базовый синтаксис

    C++14 — auto как тип возврата 
    auto add(int x, int y)
{
    return x + y; // компилятор выводит: int
}

// Все return должны возвращать один тип:
auto someFcn(bool b)
{
    if (b)
        return 5;   // int
    else
        return 6.7; // double — ОШИБКА: разные типы!
}

    Trailing return type — синтаксис с хвостовым типом

    Альтернативный синтаксис: тип возврата указывается после -> в конце объявления. Здесь auto — просто часть синтаксиса, не вывод типа.

    C++ — trailing return type 
    // Обычный синтаксис
int add(int x, int y) { return x + y; }

// Trailing return type — эквивалентно
auto add(int x, int y) -> int { return x + y; }

// Полезно для выравнивания объявлений:
auto add(int x, int y)          -> int;
auto divide(double x, double y) -> double;
auto printSomething()           -> void;

// Полезно для сложных типов:
#include <type_traits>
// Без trailing: тип возврата в начале — трудно читать
std::common_type_t<int, double> compare(int, double);
// С trailing: имя функции сразу видно
auto compare(int, double) -> std::common_type_t<int, double>;

    Ограничения вывода типа для функций

    Важное ограничение

    Функция с auto-возвратом должна быть полностью определена до первого вызова. Предварительного объявления недостаточно — компилятор не может вывести тип без тела функции.

    C++ — ограничение: нужно полное определение 
    auto foo(); // предварительное объявление

int main()
{
    foo(); // ОШИБКА: компилятор не знает тип возврата
    return 0;
}

auto foo() { return 5; } // определение — слишком поздно

    Когда использовать

    📋 Рекомендации
    • Предпочитайте явный тип возврата — он виден в объявлении функции
    • Используйте auto-возврат когда тип сложный или очевиден из контекста
    • Trailing return type полезен для выравнивания и сложных типов
    • Не используйте auto для параметров функции (до C++20 — ошибка, с C++20 — шаблоны)
