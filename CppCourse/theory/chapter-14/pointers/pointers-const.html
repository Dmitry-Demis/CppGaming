    Указатели

    Указатели и const

    С указателями const может применяться в двух местах: к типу, на который указывает указатель, и к самому указателю. Это даёт три разных комбинации.

    Правило чтения

    💡 Как читать объявление указателя

    Читайте справа налево: const int* ptr → «ptr — указатель на const int».
    int* const ptr → «ptr — const указатель на int».

    Или проще: const слева от * — нельзя изменить значение. const справа от * — нельзя изменить адрес.

    Три случая

    C++ — все варианты const с указателями
    int v { 5 };

// 1. Обычный указатель: можно всё
int* ptr0 { &v };
ptr0 = nullptr; // OK: меняем адрес
*ptr0 = 10;     // OK: меняем значение

// 2. Указатель на const: нельзя менять значение через указатель
const int* ptr1 { &v };
ptr1 = nullptr; // OK: адрес менять можно
// *ptr1 = 10;  // ОШИБКА: значение через ptr1 менять нельзя

// 3. Константный указатель: нельзя менять адрес
int* const ptr2 { &v };
// ptr2 = nullptr; // ОШИБКА: адрес менять нельзя
*ptr2 = 10;     // OK: значение менять можно

// 4. Константный указатель на const: нельзя ничего
const int* const ptr3 { &v };
// ptr3 = nullptr; // ОШИБКА
// *ptr3 = 10;     // ОШИБКА

    Сводная таблица

    Объявление Менять адрес? Менять значение?
    int* ptr ✅ Да ✅ Да
    const int* ptr ✅ Да ❌ Нет
    int* const ptr ❌ Нет ✅ Да
    const int* const ptr ❌ Нет ❌ Нет

    Передача по адресу

    C++ — передача по адресу
    #include <iostream>
#include <string>

// const: только читаем
void printByAddress(const std::string* ptr)
{
    if (ptr) std::cout << *ptr << '\n';
}

// без const: можем изменить
void appendBang(std::string* ptr)
{
    if (ptr) *ptr += "!";
}

int main()
{
    std::string s { "Привет" };
    printByAddress(&s);  // Привет
    appendBang(&s);
    printByAddress(&s);  // Привет!
    return 0;
}
    💡 Передача по адресу vs по ссылке

    В большинстве случаев предпочитайте передачу по ссылке. Передача по адресу нужна когда: нужно передать «ничего» (nullptr), нужно изменить сам указатель, или работаете с C API.
