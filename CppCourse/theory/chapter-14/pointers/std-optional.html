    std::optional

    Иногда функция не может вернуть осмысленное значение — например, деление на ноль или поиск элемента, которого нет. Как сообщить об этом вызывающему коду? C++17 предлагает элегантное решение: std::optional<T>.

    Проблема: что возвращать при ошибке

    Классические подходы имеют недостатки:

    C++ — старые подходы (проблемы) 
    // Подход 1: магическое значение-сигнал
// Проблема: -1 может быть валидным результатом
int findIndex(int val) {
    // ...
    return -1; // "не найдено" — но что если -1 — нормальный индекс?
}

// Подход 2: out-параметр
// Проблема: неудобный синтаксис, легко забыть проверить
bool divide(int x, int y, int& result) {
    if (y == 0) return false;
    result = x / y;
    return true;
}

    std::optional: чистое решение

    C++ — std::optional 
    #include <iostream>
#include <optional> // C++17

std::optional<int> divide(int x, int y)
{
    if (y == 0)
        return {};           // нет значения (std::nullopt)
    return x / y;            // есть значение
}

int main()
{
    auto r1 { divide(10, 2) };
    if (r1)                          // проверка наличия значения
        std::cout << *r1 << '\n';   // 5 — разыменование

    auto r2 { divide(10, 0) };
    if (!r2)
        std::cout << "Деление на ноль!\n";

    // value_or: значение или запасное
    std::cout << r2.value_or(-1) << '\n'; // -1

    return 0;
}

    API std::optional

    C++ — все способы работы 
    #include <optional>
#include <iostream>

int main()
{
    std::optional<int> a { 42 };       // со значением
    std::optional<int> b {};           // без значения
    std::optional<int> c { std::nullopt }; // явно без значения

    // Проверка наличия значения
    if (a.has_value()) std::cout << "a есть\n";
    if (!b)            std::cout << "b нет\n";

    // Получение значения
    std::cout << *a << '\n';           // 42 (UB если нет значения!)
    std::cout << a.value() << '\n';    // 42 (бросает исключение если нет)
    std::cout << b.value_or(0) << '\n'; // 0 (запасное значение)

    return 0;
}

    std::optional vs указатель

    Синтаксис std::optional похож на указатель, но семантика другая:

    C++ — сравнение 
    // Указатель: ссылочная семантика, может быть dangling
int* ptr { nullptr };
if (ptr) std::cout << *ptr;

// optional: семантика значения, всегда безопасно
std::optional<int> opt {};
if (opt) std::cout << *opt;

// Ключевое отличие: optional ВЛАДЕЕТ значением
// Возврат из функции — безопасно (копирует значение)
std::optional<int> getVal() {
    int local { 42 };
    return local; // OK! optional копирует значение
}
// int* getPtr() { int local{42}; return &local; } // ОПАСНО!
    Когда использовать std::optional

    Используй std::optional<T> как возвращаемый тип функции, которая может не вернуть результат. Это лучше, чем магические значения-сигналы или out-параметры.

    Не используй для параметров функций с дорогостоящим копированием — лучше const T* или перегрузка функций.
