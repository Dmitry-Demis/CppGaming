    Перегрузка функций

    Удаление перегрузок: = delete

    Иногда нужно явно запретить вызов функции с определёнными типами аргументов. Спецификатор = delete говорит компилятору: «если кто-то попытается вызвать эту версию — это ошибка компиляции».

    Проблема: нежелательные неявные преобразования

    C++ — неожиданные вызовы
    #include <iostream>

void printInt(int x) { std::cout << x << '\n'; }

int main()
{
    printInt(5);    // OK: 5
    printInt('a');  // OK? Выводит 97 — это имеет смысл?
    printInt(true); // OK? Выводит 1 — это имеет смысл?
    return 0;
}

    Решение: = delete

    C++ — запрет нежелательных типов
    #include <iostream>

void printInt(int x) { std::cout << x << '\n'; }

void printInt(char)  = delete; // запрещаем вызов с char
void printInt(bool)  = delete; // запрещаем вызов с bool

int main()
{
    printInt(97);   // OK: 97
    printInt('a');  // compile error: deleted function
    printInt(true); // compile error: deleted function
    printInt(5.0);  // compile error: ambiguous (double → int или deleted?)
    return 0;
}
    💡 = delete означает «запрещено», не «не существует»

    Удалённые функции участвуют в разрешении перегрузки. Если удалённая функция — лучшее совпадение, компилятор выдаёт ошибку. Это отличается от ситуации, когда функции вообще нет.

    Шаблонный delete: запрет всех остальных типов

    C++ — разрешить только int, запретить всё остальное
    #include <iostream>

void printInt(int x) { std::cout << x << '\n'; }

// Шаблон перехватывает все остальные типы и запрещает их
template <typename T>
void printInt(T) = delete;

int main()
{
    printInt(42);    // OK: точное совпадение с printInt(int)
    printInt('a');   // compile error: matches deleted template
    printInt(3.14);  // compile error: matches deleted template
    printInt(true);  // compile error: matches deleted template
    return 0;
}
