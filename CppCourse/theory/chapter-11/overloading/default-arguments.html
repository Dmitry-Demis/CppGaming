    Аргументы по умолчанию

    Аргумент по умолчанию — значение, которое компилятор подставляет автоматически, если вызывающий код не передал аргумент для этого параметра. Удобно, когда у параметра есть «разумное» значение по умолчанию.

    Синтаксис

    C++ — аргументы по умолчанию
    #include <iostream>

// y имеет значение по умолчанию 4
void print(int x, int y = 4)
{
    std::cout << "x=" << x << " y=" << y << '\n';
}

int main()
{
    print(1, 2); // y = 2 (явный аргумент)
    print(3);    // y = 4 (значение по умолчанию)
    // print();  // ошибка: x не имеет значения по умолчанию
    return 0;
}
// Вывод:
// x=1 y=2
// x=3 y=4

    Правила

    📋 Ключевые правила
    • Только знак =: void f(int x = 5) — OK; скобки и фигурные скобки — нет
    • Параметры с умолчаниями должны быть правее параметров без умолчаний
    • Явные аргументы всегда передаются слева направо — нельзя пропустить средний
    • Умолчание объявляется один раз — либо в объявлении, либо в определении (не в обоих)
    C++ — несколько умолчаний
    #include <iostream>

void print(int x = 10, int y = 20, int z = 30)
{
    std::cout << x << ' ' << y << ' ' << z << '\n';
}

int main()
{
    print(1, 2, 3); // 1 2 3
    print(1, 2);    // 1 2 30
    print(1);       // 1 20 30
    print();        // 10 20 30

    // print(, , 3); // ОШИБКА: нельзя пропускать аргументы
    return 0;
}

    Живое применение

    C++ — функция с разумными умолчаниями
    #include <iostream>
#include <string>

// Функция вывода с настраиваемым разделителем и переносом строки
void printList(const std::string& item, char sep = ',', bool newline = true)
{
    std::cout << item << sep;
    if (newline) std::cout << '\n';
}

int main()
{
    printList("apple");          // apple,\n
    printList("banana", ';');    // banana;\n
    printList("cherry", ' ', false); // cherry  (без переноса)
    return 0;
}
    Умолчания и перегрузка могут конфликтовать

    Если функция с умолчанием и другая перегрузка могут совпасть при одном и том же вызове — компилятор выдаст ошибку неоднозначности.

    void foo(int x = 0) {}
void foo(double d = 0.0) {}
foo(); // error: ambiguous — какую вызвать?
