    Программы из нескольких файлов

    Реальные программы не умещаются в один файл. C++ позволяет разбить код на несколько .cpp файлов, которые компилируются независимо и затем объединяются линковщиком.

    Компиляция и линковка

    Сборка C++ программы проходит два этапа:

    • Компиляция — каждый .cpp файл компилируется отдельно в объектный файл (.o / .obj). Компилятор не видит другие .cpp файлы.
    • Линковка — линковщик объединяет объектные файлы в исполняемый файл, разрешая ссылки между ними.
    Типичная структура проекта
    project/
+-- main.cpp        — точка входа
+-- math.h          — объявления функций из math.cpp
+-- math.cpp        — определения математических функций
+-- io.h            — объявления функций из io.cpp
L-- io.cpp          — определения функций ввода/вывода

    Forward declarations между файлами

    Чтобы main.cpp мог вызвать функцию из math.cpp, нужно добавить её forward declaration в main.cpp (или включить заголовочный файл). Без этого компилятор не знает о существовании функции.

    Ошибка линковщика vs компилятора

    Если вызвать функцию без forward declaration — ошибка компилятора (не знает о функции). Если forward declaration есть, но определения нет ни в одном файле — ошибка линковщика (undefined reference).

    Компиляция нескольких файлов

    Команды компиляции
    # Компилируем все .cpp файлы вместе:
g++ -std=c++17 main.cpp math.cpp io.cpp -o program

# Или по отдельности, потом линкуем:
g++ -std=c++17 -c math.cpp -o math.o
g++ -std=c++17 -c io.cpp   -o io.o
g++ -std=c++17 -c main.cpp -o main.o
g++ main.o math.o io.o -o program
