    Препроцессор

    Прежде чем компилятор увидит код, его обрабатывает препроцессор — инструмент текстовой подстановки. Он выполняет директивы, начинающиеся с #.

    #include и #define

    #include буквально вставляет содержимое файла на место директивы. #define создаёт макрос — текстовую замену. Для констант в современном C++ предпочтительнее constexpr.

    Условная компиляция

    #ifdef / #ifndef / #if позволяют включать или исключать блоки кода в зависимости от определённых макросов. Типичное применение — отладочный вывод и платформо-зависимый код.

    Препроцессор vs язык
    • Препроцессор не знает C++ — он работает с текстом до компиляции
    • Макросы не имеют типов, не отлаживаются, не уважают области видимости
    • Для констант используй constexpr, для inline-функций — inline или шаблоны
    • Оставь #define для header guards и условной компиляции
