    Заголовочные файлы

    Когда программа разбита на несколько .cpp файлов, каждому нужны forward declarations функций из других файлов. Копировать их вручную — плохая идея. Решение — заголовочные файлы (.h).

    Что писать в заголовочном файле

    ? Что помещают в .h файл
    • Forward declarations функций
    • Объявления классов и структур
    • constexpr и inline константы
    • Шаблоны (полностью — и объявление, и определение)
    • Псевдонимы типов (using, typedef)
    ? Что НЕ писать в .h файл
    • Определения обычных функций (нарушение ODR при включении в несколько файлов)
    • Определения глобальных переменных
    • using namespace std;

    Паттерн .h + .cpp

    Стандартная структура для каждого модуля:

    math.h — заголовочный файл 
    // загрузка...
    math.cpp + main.cpp — реализация и использование 
    // загрузка...

    Угловые скобки vs кавычки

    • #include <iostream> — стандартные и системные заголовки, ищутся в системных путях
    • #include "math.h" — пользовательские заголовки, ищутся сначала рядом с текущим файлом
