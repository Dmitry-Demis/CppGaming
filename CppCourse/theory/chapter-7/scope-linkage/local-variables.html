    Локальные переменные и область видимости

    В C++ нет единого атрибута «локальная переменная». Вместо этого у каждой переменной есть три независимых свойства: область видимости, время жизни и компоновка. Локальные переменные имеют конкретные значения каждого из них.

    Область видимости: блочная

    Локальные переменные имеют блочную область видимости: они доступны от точки объявления до конца блока, в котором объявлены. Попытка использовать переменную вне её области видимости — ошибка компиляции.

    C++ — блочная область видимости 
    int main()
{
    int i { 5 }; // i входит в область видимости здесь
    double d { 4.0 }; // d входит в область видимости здесь

    return 0;
} // d и i выходят из области видимости здесь

    Параметры функции тоже считаются частью блока тела функции:

    C++ — параметры функции 
    int max(int x, int y) // x и y входят в область видимости здесь
{
    int result { (x > y) ? x : y };
    return result;
} // result, y, x выходят из области видимости здесь

    Уникальность имён в одной области видимости

    В одной области видимости не может быть двух переменных с одинаковым именем:

    C++ — конфликт имён (ошибка компиляции) 
    void someFunction(int x)
{
    int x {}; // ошибка: x уже объявлен как параметр
}

    Время жизни: автоматическое

    Локальные переменные имеют автоматическое время жизни: создаются в точке объявления и уничтожаются при выходе из блока. Поэтому их иногда называют автоматическими переменными.

    C++ — автоматическое время жизни 
    int main()
{
    int i { 5 }; // i создаётся здесь
    double d { 4.0 }; // d создаётся здесь

    return 0;
} // d и i уничтожаются здесь

    Вложенные блоки и область видимости

    Переменная из внешнего блока видна во вложенном. Обратное неверно: переменная из вложенного блока недоступна снаружи.

    C++ — вложенные блоки 
    #include <iostream>

int main()
{
    int x { 5 }; // x создаётся здесь

    {
        int y { 7 }; // y создаётся здесь
        std::cout << x << " + " << y << " = " << x + y << '\n'; // x и y видны
    } // y уничтожается здесь

    // y здесь недоступен!
    return 0;
} // x уничтожается здесь

    Компоновка: отсутствует

    Локальные переменные не имеют компоновки. Это значит, что каждое объявление с одним именем в разных областях видимости — это разные, независимые объекты:

    C++ — отсутствие компоновки 
    int main()
{
    int x { 2 }; // локальная переменная, нет компоновки

    {
        int x { 3 }; // другой объект, несмотря на то же имя
    }

    return 0;
}

    Объявляйте переменные в минимальной области видимости

    Чем меньше область видимости переменной, тем проще понять программу. Если переменная нужна только внутри блока — объявляйте её там:

    C++ — минимальная область видимости 
    #include <iostream>

int main()
{
    {
        int y { 5 }; // y нужен только здесь
        std::cout << y << '\n';
    }
    // y здесь недоступен — это хорошо
    return 0;
}
    Лучшая практика

    Объявляйте переменные в наименьшей подходящей области видимости. Не создавайте блоки только ради ограничения области видимости — лучше вынесите код в отдельную функцию.
