    Пространства имён и оператор ::

    Когда программа растёт, вероятность конфликта имён растёт экспоненциально: каждый новый идентификатор может столкнуться с любым из уже существующих. Пространство имён — это именованная область, которая изолирует идентификаторы и устраняет конфликты.

    Проблема конфликта имён

    Представьте два файла с одинаковой функцией:

    C++ — конфликт имён (ошибка компоновщика) 
    // foo.cpp
int doSomething(int x, int y) { return x + y; }

// goo.cpp
int doSomething(int x, int y) { return x - y; } // ошибка: переопределение!

    Компоновщик выдаст ошибку: два определения одного символа. Решение — пространства имён.

    Объявление пространства имён

    C++ — пространства имён устраняют конфликт 
    // foo.cpp
namespace Foo
{
    int doSomething(int x, int y) { return x + y; }
}

// goo.cpp
namespace Goo
{
    int doSomething(int x, int y) { return x - y; }
}

// main.cpp
#include <iostream>
int main()
{
    std::cout << Foo::doSomething(4, 3) << '\n'; // 7
    std::cout << Goo::doSomething(4, 3) << '\n'; // 1
    return 0;
}

    Оператор разрешения области видимости ::

    Оператор :: говорит компилятору: «ищи идентификатор справа в области видимости слева». Если слева ничего нет — ищи в глобальном пространстве имён.

    C++ — оператор :: 
    std::cout   // cout из пространства std
::foo       // foo из глобального пространства имён
Foo::bar    // bar из пространства Foo

    Вложенные пространства имён

    Пространства имён можно вкладывать. С C++17 есть удобный синтаксис через :::

    C++ — вложенные пространства имён (C++17) 
    #include <iostream>

namespace Foo::Goo // C++17: Goo вложен в Foo
{
    int add(int x, int y) { return x + y; }
}

int main()
{
    std::cout << Foo::Goo::add(1, 2) << '\n'; // 3
    return 0;
}
    📌 До C++17

    Вложенные пространства имён писались явно:

    namespace Foo { namespace Goo { int add(int x, int y); } }

    Псевдонимы пространств имён

    Длинные вложенные имена можно сократить псевдонимом:

    C++ — псевдоним пространства имён 
    namespace Active = Foo::Goo; // Active — псевдоним для Foo::Goo

std::cout << Active::add(1, 2) << '\n'; // то же, что Foo::Goo::add

    Предварительные объявления в пространствах имён

    Предварительное объявление должно находиться в том же пространстве имён, что и определение:

    C++ — forward declaration в namespace 
    // add.h
namespace BasicMath
{
    int add(int x, int y); // объявление
}

// add.cpp
#include "add.h"
namespace BasicMath
{
    int add(int x, int y) { return x + y; } // определение
}

// main.cpp
#include "add.h"
#include <iostream>
int main()
{
    std::cout << BasicMath::add(4, 3) << '\n'; // 7
}
    Лучшая практика

    Не добавляйте ничего в пространство имён std — это неопределённое поведение.
