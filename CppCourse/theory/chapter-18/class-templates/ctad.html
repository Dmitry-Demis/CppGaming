    CTAD — вывод аргументов шаблона класса

    Начиная с C++17, компилятор может автоматически выводить аргументы шаблона класса из инициализатора объекта. Это называется Class Template Argument Deduction (CTAD).

    Базовый пример

    C++17
    #include <utility>

int main()
{
    std::pair<int, int> p1 { 1, 2 }; // явно (C++11)
    std::pair p2 { 1, 2 };           // CTAD: выводит std::pair<int, int> (C++17)
    return 0;
}

    Deduction guides

    Для пользовательских агрегатных шаблонов в C++17 нужно явно написать deduction guide. В C++20 это делается автоматически.

    C++17
    template <typename T, typename U>
struct Pair { T first{}; U second{}; };

// Deduction guide (нужен только в C++17)
template <typename T, typename U>
Pair(T, U) -> Pair<T, U>;

int main()
{
    Pair p { 1, 2.3 }; // выводит Pair<int, double>
    return 0;
}

    Параметры шаблона по умолчанию

    C++
    template <typename T = int, typename U = int>
struct Pair { T first{}; U second{}; };

template <typename T, typename U>
Pair(T, U) -> Pair<T, U>;

int main()
{
    Pair p1 { 1, 2 };  // Pair<int, int> через CTAD
    Pair p2;           // Pair<int, int> через defaults
    return 0;
}
    ⚠️ Ограничения CTAD
    • Не работает в non-static member initialization
    • Не работает в параметрах функций (используйте шаблон функции)
    • Требует полного списка аргументов или их отсутствия — частичный список не допускается
