    Глобальные константы в нескольких файлах

    Когда одни и те же константы нужны в нескольких файлах (например, pi, гравитация, коэффициенты), их нужно объявить один раз и использовать везде. Есть три подхода.

    Способ 1: константы как внутренние переменные (до C++17)

    Самый простой: определяем constexpr в заголовочном файле.

    C++ — constants.h (внутренние) 
    #ifndef CONSTANTS_H
#define CONSTANTS_H

namespace constants
{
    constexpr double pi { 3.14159 };
    constexpr double avogadro { 6.022e23 };
    constexpr double gravity { 9.8 };
}
#endif

    При включении в несколько .cpp файлов каждый получает свою копию констант. Компилятор обычно оптимизирует их, но при изменении заголовка нужно перекомпилировать все файлы.

    Способ 2: константы как внешние переменные (до C++17)

    Одна копия в .cpp файле, предварительные объявления в заголовке:

    C++ — constants.cpp + constants.h (внешние) 
    // constants.cpp
#include "constants.h"
namespace constants
{
    extern constexpr double pi { 3.14159 };
    extern constexpr double gravity { 9.8 };
}

// constants.h
#ifndef CONSTANTS_H
#define CONSTANTS_H
namespace constants
{
    extern const double pi;    // объявление (не constexpr!)
    extern const double gravity;
}
#endif
    Ограничение

    Через предварительное объявление константы не являются constexpr — их нельзя использовать в контекстах, требующих константного выражения (например, размер массива) вне файла определения.

    Способ 3: inline constexpr (C++17, рекомендуется)

    Лучший вариант: inline constexpr в заголовке. Одна копия в программе, доступна как константное выражение везде:

    C++ — constants.h (inline constexpr, C++17) 
    #ifndef CONSTANTS_H
#define CONSTANTS_H

namespace constants
{
    inline constexpr double pi { 3.14159 };      // C++17
    inline constexpr double avogadro { 6.022e23 };
    inline constexpr double gravity { 9.8 };
}
#endif

// main.cpp
#include "constants.h"
#include <iostream>

int main()
{
    double radius { 5.0 };
    std::cout << 2 * radius * constants::pi << '\n';
    return 0;
}

    Сравнение подходов

    📊 Сравнение трёх способов
    Способ Работает до C++17 Одна копия constexpr везде
    Внутренние (заголовок) ✅ ❌ (копия в каждом .cpp) ✅
    Внешние (.cpp + .h) ✅ ✅ ❌ (только в .cpp)
    inline constexpr (C++17) ❌ ✅ ✅
    Лучшая практика

    Если компилятор поддерживает C++17 — используйте inline constexpr в заголовочном файле.
