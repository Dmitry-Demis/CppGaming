    Статические локальные переменные

    Ключевое слово static применительно к локальной переменной меняет её время жизни с автоматического на статическое: переменная создаётся при запуске программы и уничтожается при её завершении. При этом область видимости остаётся блочной.

    Автоматическая vs статическая локальная

    C++ — автоматическая (сбрасывается при каждом вызове) 
    #include <iostream>

void incrementAndPrint()
{
    int value { 1 }; // создаётся заново при каждом вызове
    ++value;
    std::cout << value << '\n';
}

int main()
{
    incrementAndPrint(); // 2
    incrementAndPrint(); // 2
    incrementAndPrint(); // 2
    return 0;
}
    C++ — статическая (сохраняет значение между вызовами) 
    #include <iostream>

void incrementAndPrint()
{
    static int s_value { 1 }; // инициализируется только один раз
    ++s_value;
    std::cout << s_value << '\n';
} // s_value не уничтожается, но выходит из области видимости

int main()
{
    incrementAndPrint(); // 2
    incrementAndPrint(); // 3
    incrementAndPrint(); // 4
    return 0;
}

    Практический пример: генератор уникальных ID

    Статические локальные переменные идеальны для генераторов уникальных идентификаторов:

    C++ — генератор ID 
    int generateID()
{
    static int s_itemID { 0 };
    return s_itemID++; // возвращает текущее значение, затем увеличивает
}

// generateID() → 0
// generateID() → 1
// generateID() → 2 ...

    Диаграмма: область видимости vs время жизни

    📐 Статическая локальная переменная
    Время жизни:  ├──────────────────────────────────────────┤
              Запуск программы              Завершение программы

Область       ├──────┤  ├──────┤  ├──────┤
видимости:    Вызов 1   Вызов 2   Вызов 3
              (внутри функции)

    Не используйте для управления потоком

    Статические локальные переменные делают функцию непереиспользуемой, если они влияют на её поведение. Лучше передавать состояние через параметры:

    C++ — плохо: static меняет поведение функции 
    // Плохо: функция ведёт себя по-разному при первом и последующих вызовах
int getInteger()
{
    static bool s_isFirstCall { true };
    if (s_isFirstCall) { std::cout << "Введите число: "; s_isFirstCall = false; }
    else { std::cout << "Введите ещё число: "; }
    int i {}; std::cin >> i; return i;
}

// Хорошо: поведение управляется параметром
int getInteger(bool isFirstCall)
{
    if (isFirstCall) std::cout << "Введите число: ";
    else std::cout << "Введите ещё число: ";
    int i {}; std::cin >> i; return i;
}
    Лучшая практика

    const статические локальные переменные — нормально. Неконстантные — избегайте, если только переменная не уникальна во всей программе и никогда не нуждается в сбросе.
