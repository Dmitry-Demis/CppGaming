    Внутренняя компоновка

    Компоновка (linkage) определяет, ссылаются ли несколько объявлений одного имени в разных областях видимости на один и тот же объект. Идентификатор с внутренней компоновкой виден только в пределах одной единицы трансляции (одного .cpp-файла) и недоступен из других файлов.

    Единица трансляции

    Единица трансляции — это один .cpp-файл вместе со всеми включёнными в него заголовочными файлами. Компилятор обрабатывает каждую единицу трансляции независимо.

    📐 Диаграмма компоновки
    ┌─────────────────────────────────────────────────────┐
│                    Программа                        │
│  ┌──────────────────┐    ┌──────────────────┐       │
│  │   a.cpp (ЕТ 1)   │    │  main.cpp (ЕТ 2) │       │
│  │                  │    │                  │       │
│  │ static int g_x   │    │ static int g_x   │       │
│  │ (внутренняя)     │    │ (внутренняя)     │       │
│  │ — своя копия     │    │ — своя копия     │       │
│  └──────────────────┘    └──────────────────┘       │
│                                                     │
│  Компоновщик НЕ видит переменные с внутр. компоновкой│
└─────────────────────────────────────────────────────┘

    Ключевое слово static для глобальных переменных

    Чтобы сделать глобальную переменную с внутренней компоновкой, используют static:

    C++ — внутренняя компоновка через static 
    #include <iostream>

static int g_x {};       // внутренняя компоновка (явно)
const int g_y { 1 };     // внутренняя компоновка по умолчанию
constexpr int g_z { 2 }; // внутренняя компоновка по умолчанию

int main()
{
    std::cout << g_x << ' ' << g_y << ' ' << g_z << '\n';
    return 0;
}
    💡 Ключевой момент

    const и constexpr глобальные переменные имеют внутреннюю компоновку по умолчанию — static для них не нужен.

    Пример: два файла с одинаковым именем

    C++ — независимые g_x в двух файлах 
    // a.cpp
[[maybe_unused]] constexpr int g_x { 2 }; // только в a.cpp

// main.cpp
#include <iostream>
static int g_x { 3 }; // только в main.cpp

int main()
{
    std::cout << g_x << '\n'; // 3 — из main.cpp
    return 0;
}

    Функции с внутренней компоновкой

    Функции тоже можно сделать «приватными» для файла через static:

    C++ — static-функция 
    // helper.cpp
static int add(int x, int y) // недоступна из других файлов
{
    return x + y;
}

    Краткий справочник

    C++ — синтаксис внутренней компоновки 
    static int g_x;          // неинициализированная внутренняя глобальная
static int g_x { 1 };    // инициализированная внутренняя глобальная

const int g_y { 2 };     // внутренняя const глобальная
constexpr int g_y { 3 }; // внутренняя constexpr глобальная

static int foo() {}      // внутренняя функция
    Лучшая практика

    Давайте внутреннюю компоновку всем идентификаторам, которые не должны быть доступны из других файлов. Для группы таких идентификаторов удобнее использовать безымянное пространство имён (см. следующий параграф).
