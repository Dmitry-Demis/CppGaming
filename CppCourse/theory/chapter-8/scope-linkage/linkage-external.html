    Внешняя компоновка

    Идентификатор с внешней компоновкой виден и используется как в файле, где он определён, так и в других файлах программы — через предварительное объявление. Именно поэтому такие идентификаторы по-настоящему «глобальны».

    Функции имеют внешнюю компоновку по умолчанию

    Вы уже пользовались этим: функция из одного .cpp-файла вызывается из другого через предварительное объявление.

    C++ — функция с внешней компоновкой 
    // a.cpp
#include <iostream>
void sayHi() // внешняя компоновка по умолчанию
{
    std::cout << "Hi!\n";
}

// main.cpp
void sayHi(); // предварительное объявление

int main()
{
    sayHi(); // компоновщик найдёт определение в a.cpp
    return 0;
}

    Глобальные переменные с внешней компоновкой

    Неконстантные глобальные переменные имеют внешнюю компоновку по умолчанию. Чтобы использовать их в другом файле, нужно предварительное объявление через extern:

    C++ — extern для переменных 
    // a.cpp — определения
int g_x { 2 };              // внешняя компоновка по умолчанию
extern const int g_y { 3 }; // const нужно явно пометить extern

// main.cpp — предварительные объявления
#include <iostream>
extern int g_x;       // объявление: g_x определён где-то ещё
extern const int g_y; // объявление: g_y определён где-то ещё

int main()
{
    std::cout << g_x << ' ' << g_y << '\n'; // 2 3
    return 0;
}

    Два значения extern

    ⚠️ Важно: extern имеет два смысла
    • extern const int g_y { 3 }; — определение с внешней компоновкой
    • extern const int g_y; — предварительное объявление (без инициализатора)

    Краткий справочник

    C++ — синтаксис внешней компоновки 
    // Предварительные объявления (extern без инициализатора):
extern int g_y;             // для неконстантной глобальной
extern const int g_y;       // для const глобальной
// extern constexpr int g_y; — НЕЛЬЗЯ: constexpr нельзя forward-declare

// Определения внешних переменных:
int g_x;                    // неконстантная (extern по умолчанию)
int g_x { 1 };              // неконстантная с инициализатором
extern const int g_x { 2 }; // const с внешней компоновкой
    Лучшая практика

    Используйте extern только для предварительных объявлений глобальных переменных или для определений const глобальных переменных с внешней компоновкой. Не добавляйте extern к неконстантным определениям — они и так внешние.
