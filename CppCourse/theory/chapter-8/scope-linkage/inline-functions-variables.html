    Inline-функции и переменные

    Слово inline в современном C++ означает не «встрой вызов функции», а «разрешено иметь несколько одинаковых определений». Это ключевое отличие от исторического смысла.

    Встраивание вызовов (историческое)

    Исторически inline было подсказкой компилятору заменить вызов функции её телом. Это устраняет накладные расходы на вызов для маленьких функций:

    C++ — встраивание вызова 
    // Исходный код:
int min(int x, int y) { return (x < y) ? x : y; }
std::cout << min(5, 6) << '\n';

// После встраивания компилятор видит примерно так:
std::cout << ((5 < 6) ? 5 : 6) << '\n';
// И может дополнительно оптимизировать до:
std::cout << 5 << '\n';
    💡 Современная реальность

    Современные компиляторы сами решают, встраивать ли вызов. Ключевое слово inline как подсказка для встраивания игнорируется. Не используйте его с этой целью.

    Современный смысл: несколько определений

    Сегодня inline означает: «это определение может встречаться в нескольких единицах трансляции, и это не нарушение ODR». Компоновщик оставит одну копию.

    Это позволяет определять функции прямо в заголовочных файлах:

    C++ — inline-функция в заголовке 
    // pi.h
#ifndef PI_H
#define PI_H

inline double pi() { return 3.14159; }

#endif

// main.cpp и math.cpp оба включают pi.h
// Компоновщик дедуплицирует определения — нет нарушения ODR

    Inline-переменные (C++17)

    C++17 добавил inline для переменных. Это позволяет определять глобальные переменные в заголовочных файлах без нарушения ODR:

    C++ — inline-переменная в заголовке (C++17) 
    // constants.h
namespace constants
{
    inline constexpr double pi { 3.14159 };    // C++17
    inline constexpr double avogadro { 6.022e23 };
}

// Можно включать в любое количество .cpp файлов —
// переменная будет существовать в одном экземпляре
    📌 Неявно inline

    Следующие сущности неявно являются inline:

    • Функции, определённые внутри класса/структуры/объединения
    • constexpr/consteval функции
    • Функции, инстанцированные из шаблонов

    constexpr переменные — НЕ неявно inline!

    Лучшая практика

    Не используйте inline для запроса встраивания. Используйте его только когда определяете функции или переменные в заголовочных файлах.
