    Массивы и циклы

    Массивы и циклы — неразрывная пара. Цикл позволяет обойти все элементы массива, не перечисляя их вручную. Это делает код масштабируемым: он работает одинаково для 5 и для 5000 элементов.

    Базовый обход

    C++ — среднее значение через цикл 
    #include <iostream>
#include <vector>

int main()
{
    std::vector testScore { 84, 92, 76, 81, 56 };
    std::size_t length { testScore.size() };

    int sum { 0 };
    for (std::size_t i { 0 }; i < length; ++i) // i от 0 до length-1
        sum += testScore[i];

    int average { sum / static_cast<int>(length) };
    std::cout << "Среднее: " << average << '\n'; // 77

    return 0;
}
    Типичная ошибка: off-by-one

    Условие i <= length вместо i < length — выход за границы и неопределённое поведение. Последний допустимый индекс — length - 1.

    Что можно делать при обходе

    C++ — поиск максимума 
    #include <iostream>
#include <vector>

int main()
{
    std::vector data { 84, 92, 76, 81, 56 };

    int maxVal { data[0] }; // начинаем с первого элемента
    for (std::size_t i { 1 }; i < data.size(); ++i)
    {
        if (data[i] > maxVal)
            maxVal = data[i];
    }

    std::cout << "Максимум: " << maxVal << '\n'; // 92
    return 0;
}

    Шаблон функции + массив + цикл = масштабируемость

    Комбинация шаблонов, массивов и циклов позволяет писать код, который работает с любым типом элементов и любым количеством:

    C++ — шаблон функции для среднего 
    #include <iostream>
#include <vector>

template <typename T>
T calculateAverage(const std::vector<T>& arr)
{
    std::size_t length { arr.size() };
    T sum { 0 };
    for (std::size_t i { 0 }; i < length; ++i)
        sum += arr[i];
    return sum / static_cast<int>(length);
}

int main()
{
    std::vector class1 { 84, 92, 76, 81, 56 };
    std::cout << calculateAverage(class1) << '\n'; // 77 (int)

    std::vector class2 { 93.2, 88.6, 64.2, 81.0 };
    std::cout << calculateAverage(class2) << '\n'; // 81.75 (double)

    return 0;
}

    Передача вектора в функцию

    Вектор передаётся по const-ссылке, чтобы избежать дорогого копирования:

    C++ — передача по const-ссылке 
    void printArray(const std::vector<int>& arr) // const ref — без копирования
{
    for (std::size_t i { 0 }; i < arr.size(); ++i)
        std::cout << arr[i] << ' ';
    std::cout << '\n';
}

// Шаблонная версия — работает с любым типом элементов:
template <typename T>
void printArray(const std::vector<T>& arr)
{
    for (std::size_t i { 0 }; i < arr.size(); ++i)
        std::cout << arr[i] << ' ';
    std::cout << '\n';
}
    💡 Возврат вектора по значению

    В отличие от передачи, возвращать вектор по значению — нормально. Компилятор применяет оптимизацию (copy elision или move semantics), и копирования не происходит.

    std::vector<int> makeVector() {
    std::vector<int> result { 1, 2, 3 };
    return result; // эффективно: move или elision
}
