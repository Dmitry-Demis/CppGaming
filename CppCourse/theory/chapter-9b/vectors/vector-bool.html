    Массивы и std::vector

    std::vector<bool> и его особенности

    std::vector<bool> — особый случай. Стандарт разрешает реализации упаковывать 8 булевых значений в один байт для экономии памяти. Это звучит хорошо, но создаёт неожиданные проблемы.

    Базовое использование

    C++ — std::vector<bool> 
    #include <iostream>
#include <vector>

int main()
{
    std::vector<bool> v { true, false, false, true, true };

    for (int i : v)
        std::cout << i << ' '; // 1 0 0 1 1

    v[4] = false;

    for (int i : v)
        std::cout << i << ' '; // 1 0 0 1 0

    return 0;
}

    Проблемы

    std::vector<bool> — это не настоящий вектор. Он не хранит bool значения, не гарантирует непрерывность в памяти и не является полноценным контейнером стандартной библиотеки. Код, который работает с любым другим типом, может не работать с bool:

    C++ — ломается с bool, работает с другими типами 
    template<typename T>
void foo(std::vector<T>& v)
{
    T& first = v[0]; // работает для int, double, string...
                     // НЕ работает для bool!
}

std::vector<int>  vi { 1, 2, 3 };
std::vector<bool> vb { true, false };

foo(vi); // OK
foo(vb); // ошибка компиляции!

    Альтернативы

    Тип Когда использовать
    std::vector<char> Нужен изменяемый массив булевых значений — это настоящий контейнер
    std::bitset<N> Размер известен на этапе компиляции, нужны битовые операции
    std::vector<bool> Только если нужна экономия памяти и вы знаете об ограничениях
    C++ — рекомендуемая альтернатива 
    // Вместо std::vector<bool>:
std::vector<char> flags { 1, 0, 0, 1, 1 }; // настоящий контейнер

// Или если размер фиксирован:
#include <bitset>
std::bitset<5> bits { 0b10011 }; // битовые операции: &, |, ^, ~
    Лучшая практика

    Предпочитайте std::vector<char> или std::bitset вместо std::vector<bool>.
