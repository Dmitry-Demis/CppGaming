    Перечисления

    Scoped enumerations (enum class)

    Unscoped-перечисления не являются типобезопасными: компилятор может сравнить перечислители из разных типов, неявно преобразовав их в int. enum class решает эту проблему.

    Проблема unscoped enum

    C++ — небезопасное сравнение
    #include <iostream>
int main()
{
    enum Color { red, blue };
    enum Fruit { banana, apple };

    Color color { red };
    Fruit fruit { banana };

    if (color == fruit) // компилятор сравнивает как int: оба == 0
        std::cout << "равны\n"; // выводится!
    return 0;
}

    enum class

    enum class имеет две ключевые особенности: перечислители не преобразуются в int неявно, и они находятся в собственной области видимости.

    C++ — enum class
    #include <iostream>
int main()
{
    enum class Color { red, blue };
    enum class Fruit { banana, apple };

    Color color { Color::red };   // обязательно с префиксом
    Fruit fruit { Fruit::banana };

    // if (color == fruit) // ошибка компиляции: разные типы
    if (color == Color::red)
        std::cout << "Красный\n";
    return 0;
}

    Преобразование в int

    Неявного преобразования нет. Используйте static_cast или (C++23) std::to_underlying():

    C++
    #include <iostream>
#include <utility> // std::to_underlying (C++23)

enum class Color { red, blue };

int main()
{
    Color c { Color::blue };
    // std::cout << c;                          // ошибка
    std::cout << static_cast<int>(c) << '\n';  // 1
    std::cout << std::to_underlying(c) << '\n'; // 1 (C++23)
    return 0;
}

    using enum (C++20)

    Внутри функции можно импортировать перечислители в текущую область видимости:

    C++20
    #include <string_view>
enum class Color { black, red, blue };

constexpr std::string_view getColor(Color color)
{
    using enum Color; // импортируем перечислители
    switch (color)
    {
    case black: return "black"; // без Color::
    case red:   return "red";
    case blue:  return "blue";
    default:    return "???";
    }
}
    Лучшая практика

    Предпочитайте enum class обычному enum, если нет веских причин использовать unscoped-перечисление.
