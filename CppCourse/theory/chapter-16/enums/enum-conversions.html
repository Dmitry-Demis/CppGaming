    Перечисления

    Целочисленные преобразования enum

    Каждый перечислитель хранит целочисленное значение. По умолчанию первый перечислитель получает значение 0, каждый следующий — на 1 больше предыдущего.

    Значения перечислителей

    C++ — значения по умолчанию и явные
    enum Color
{
    black,   // 0
    red,     // 1
    blue,    // 2
    green,   // 3
};

enum Animal
{
    cat = -3,   // -3
    dog,        // -2
    pig,        // -1
    horse = 5,
    giraffe = 5, // то же значение, что и horse
    chicken,    // 6
};
    Лучшая практика

    Не задавайте явные значения перечислителям без необходимости. Перечислитель с значением 0 должен представлять наиболее логичное «по умолчанию» состояние.

    Неявное преобразование в int

    Unscoped-перечисление неявно преобразуется в целое число. Именно поэтому std::cout << shirt выводит число, а не имя:

    C++
    #include <iostream>
enum Color { black, red, blue };

int main()
{
    Color shirt { blue };
    std::cout << shirt << '\n'; // выводит 2
    return 0;
}

    Underlying type

    Тип, используемый для хранения значений перечислителей, называется underlying type. По умолчанию компилятор выбирает его сам (обычно int). Можно задать явно:

    C++ — явный underlying type
    #include <cstdint>
#include <iostream>

enum Color : std::int8_t  // 1 байт
{
    black, red, blue,
};

int main()
{
    Color c { black };
    std::cout << sizeof(c) << '\n'; // 1
    return 0;
}

    Преобразование int → enum

    Компилятор не преобразует целое число в перечисление неявно. Нужен static_cast:

    C++
    enum Pet { cat, dog, pig, whale };

int main()
{
    // Pet p { 2 };                  // ошибка компиляции
    Pet p { static_cast<Pet>(2) };  // OK: pig
    return 0;
}

    В C++17, если у перечисления задан явный underlying type, допускается list-инициализация целым числом:

    C++17
    enum Pet : int { cat, dog, pig, whale };

int main()
{
    Pet p { 2 }; // OK в C++17 при наличии явного base
    return 0;
}
