    Введение в фундаментальные типы данных

    Каждая переменная в C++ имеет тип — точное описание того, какие значения она может хранить, сколько памяти занимает и какие операции к ней применимы. Фундаментальные типы — это нулевой уровень системы типов C++: они встроены прямо в компилятор, не требуют подключения заголовков и являются строительным материалом для всех остальных типов программы.

    В отличие от языков с динамической типизацией (Python, JavaScript), C++ является статически типизированным языком: тип каждого выражения и переменной известен на этапе компиляции. Это позволяет компилятору генерировать максимально эффективный машинный код и выявлять ошибки до запуска программы.

    Что такое фундаментальные типы

    Фундаментальные типы (fundamental types) — встроенные типы языка C++, представляющие базовые значения: целые числа, числа с плавающей точкой, символы и логические значения. Они заданы непосредственно стандартом языка и не зависят от пользовательских определений или библиотек.

    Стандарт ISO/IEC 14882: [basic.fundamental] §6.7.1

    Стандарт C++ разделяет фундаментальные типы на пять групп:

    • Целочисленные типы — signed char, short int, int, long int, long long int и их беззнаковые варианты.
    • Типы с плавающей точкой — float, double, long double. Представляют вещественные числа согласно IEEE 754.
    • Символьные типы — char, signed char, unsigned char, wchar_t, , , .
    • Логический тип — bool, принимает значения true или false.
    • Тип void — специальный неполный тип, обозначающий отсутствие значения.
    ISO/IEC 14882 — [basic.fundamental] §6.7.1 «There are five standard signed integer types: "signed char", "short int", "int", "long int", and "long long int". In this list, each type provides at least as much storage as those preceding it in the list.»

    Система типов C++

    C++ — строго и статически типизированный язык. Это означает, что:

    • Каждое выражение имеет тип, определяемый на этапе компиляции.
    • Компилятор проверяет совместимость типов при каждой операции.
    • Неявные преобразования разрешены только между совместимыми типами.
    • Явное преобразование типов (static_cast, reinterpret_cast) всегда под контролем программиста.
    Статическая типизация и производительность

    Знание типов на этапе компиляции позволяет компилятору выбирать оптимальные машинные инструкции: для int — 32-битные целочисленные операции, для double — операции FPU/SIMD, для bool — битовые инструкции. Именно поэтому C++ даёт производительность, сравнимую с ассемблером.

    ? Неявные преобразования типов

    C++ разрешает неявные преобразования между числовыми типами, но они могут приводить к потере данных. Например, присваивание int в short может вызвать переполнение: значение 40000 не помещается в short (максимум 32 767). Используйте static_cast<T>() для явного преобразования типов.

    ? Знаковость char: платформенная неопределённость

    Знаковость простого char не определена стандартом — она зависит от платформы. На x86 обычно signed, на некоторых ARM — unsigned. Используйте signed char или unsigned char явно, когда знаковость важна.

    C++ 
    #include <iostream>
#include <limits>

int main() {
    signed   int si{-1};
    unsigned int ui{static_cast<unsigned int>(si)}; // все биты = 1

    std::cout << "signed int -1   = " << si << '\n';
    std::cout << "unsigned int -1 = " << ui << '\n'; // 4294967295

    // Опасность смешения знаковых и беззнаковых
    unsigned int umax{std::numeric_limits<unsigned int>::max()};
    if (-1 < umax) {
        std::cout << "Сравнение верно\n";
    } else {
        std::cout << "Ловушка: -1 неявно стал UMAX!\n";
    }

    // Знаковость char
    signed char   sc{127};
    unsigned char uc{127};
    ++sc; // UB! переполнение signed
    ++uc; // OK: 128
    std::cout << "signed char 127+1   = " << static_cast<int>(sc) << '\n';
    std::cout << "unsigned char 127+1 = " << static_cast<int>(uc) << '\n';
    return 0;
}
    Рекомендация: используйте <cstdint>

    Когда нужен тип с точно заданным размером (бинарные протоколы, оборудование), используйте int8_t, int16_t, int32_t, int64_t и uint*_t из <cstdint>. Они гарантируют точный размер на любой поддерживаемой платформе.
