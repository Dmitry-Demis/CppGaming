    Знаковые и беззнаковые типы

    Каждый целочисленный тип существует в двух вариантах: знаковый (signed) хранит отрицательные и положительные числа, а беззнаковый (unsigned) — только неотрицательные, но с удвоенным максимумом. Понимать разницу и её последствия критически важно.

    [basic.fundamental] §6.7.1 — ISO/IEC 14882:2020 Unsigned integers shall obey the laws of arithmetic modulo $2^n$, where $n$ is the number of bits in the value representation.

    Диапазоны: signed vs unsigned

    При переходе от signed к unsigned один бит перестаёт кодировать знак и начинает кодировать значение — диапазон сдвигается в ноль:

    Неявные преобразования и ловушка

    В смешанных выражениях (знаковый + беззнаковый) C++ применяет usual arithmetic conversions: знаковый операнд приводится к беззнаковому типу. Это источник классических багов:

    ? ?1 становится UINT_MAX при сравнении с unsigned

    Отрицательное значение -1 при приведении к unsigned int становится 4 294 967 295 (UINT_MAX), потому что $-1 \equiv 2^{32} - 1 \pmod{2^{32}}$. Компилятор сравнивает два беззнаковых числа, и 4294967295 > 0 — ошибка!

    C++ — ловушка signed/unsigned 
    #include <iostream>
#include <limits>
#include <vector>

int main() {
    unsigned int umax{std::numeric_limits<unsigned int>::max()};

    // ? Ловушка №1: -1 приводится к UINT_MAX
    if (-1 > umax)                    // -1 > unsigned = UINT_MAX > true!
        std::cout << "Кажется, -1 больше UINT_MAX?!\n";

    // ? Ловушка №2: цикл с unsigned счётчиком может зациклиться
    // for (unsigned i = 10; i >= 0; --i) { ... } // вечный цикл!

    // ? Ловушка №3: size() возвращает size_t (unsigned)
    std::vector<int> v{1, 2, 3};
    int x{5};
    // if (x - v.size() > 0) — UB, если x < v.size()!
    // Правильно: (size_t)x > v.size()  ИЛИ  (int)v.size() > x

    // ? Исправление: явное приведение к знаковому
    if (static_cast<long long>(-1) < static_cast<long long>(umax))
        std::cout << "OK: -1 < UINT_MAX\n";

    // ? C++20: std::cmp_less — безопасное сравнение разных целых типов
    // if (std::cmp_less(-1, umax)) { ... }

    return 0;
}

    Когда использовать unsigned?

    Несмотря на ловушки, unsigned типы полезны в ряде сценариев:

    ? Используйте unsigned когда:
    • Битовые операции — сдвиги, маски, XOR (знаковый сдвиг вправо — UB на отрицательных)
    • Бинарные форматы — байты данных, пиксели, аппаратные регистры
    • Размеры и индексы — std::size_t для .size() и .length()
    • Wrap-around нужен намеренно — CRC, хеши, криптография
    • Значение всегда неотрицательно по смыслу — например, порт TCP/IP
    ? Избегайте unsigned когда:

    Значение может участвовать в вычитании или сравнении со знаковыми переменными — лучше использовать int и проверять диапазон отдельно. Распространённая ошибка: счётчик цикла for (unsigned i = n; i >= 0; --i) — это бесконечный цикл, потому что unsigned никогда не становится отрицательным.

    C++ — правильные паттерны 
    #include <iostream>
#include <cstdint>
#include <vector>

int main() {
    // ? Битовые операции — используем unsigned
    uint32_t flags{0u};
    flags |= (1u << 3);   // установить бит 3
    flags &= ~(1u << 3);  // сбросить бит 3
    bool bit3{(flags >> 3) & 1u};

    // ? Правильный счётчик вниз (signed)
    for (int i{10}; i >= 0; --i)
        std::cout << i << ' ';
    std::cout << '\n';

    // ? size_t для индексов контейнера
    std::vector<int> v{10, 20, 30};
    for (std::size_t i{0}; i < v.size(); ++i)
        std::cout << v[i] << ' ';
    std::cout << '\n';

    // ? range-for — избегает проблемы с типами вообще
    for (int val : v)
        std::cout << val << ' ';
    std::cout << '\n';

    return 0;
}
