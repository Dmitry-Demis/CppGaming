    Переполнение целых чисел

    Целые числа хранятся в фиксированном числе битов. Если результат вычисления выходит за пределы диапазона типа — происходит переполнение. Поведение зависит от того, знаковый тип или беззнаковый — и разница принципиальная.

    Знаковое переполнение — UB

    Переполнение signed — неопределённое поведение (Undefined Behaviour)

    Стандарт C++ явно указывает: переполнение знакового целого — это UB. Компилятор не обязан ничего делать — он вправе сгенерировать любой код, удалить «невозможные» ветки или устроить crash.

    Оптимизаторы активно используют этот факт: они предполагают, что UB никогда не случается, и на этой основе упрощают код. Это делает переполнение signed особенно коварным: оно может работать в debug-сборке и молча ломаться в release.

    INT_MAX = 2 147 483 647 +1 > ?? (UB) ? ?2 147 483 648
    C++ — signed overflow (UB) 
    #include <iostream>
#include <limits>

int main() {
    int max{std::numeric_limits<int>::max()};

    // ? UB: компилятор может делать что угодно
    // int overflow = max + 1;

    // Реальный пример: компилятор может УБРАТЬ эту проверку!
    // if (max + 1 > max) — оптимизатор знает, что overflow UB,
    // значит max + 1 > max всегда "истина" > убирает else ветку

    // ? Безопасно: проверяем ДО операции
    if (max < std::numeric_limits<int>::max()) {
        int result{max + 1};  // теперь безопасно, мы проверили
        std::cout << result << '\n';
    } else {
        std::cout << "Переполнение предотвращено!\n";
    }

    return 0;
}

    Беззнаковое переполнение — wrap-around

    Для беззнаковых типов переполнение строго определено стандартом: результат берётся по модулю $2^n$, где $n$ — число бит типа. Это называется wrap-around (обёртка вокруг нуля).

    UINT_MAX = 4 294 967 295 +1 > 0 (определено!)
    0u ?1 > 4 294 967 295 (определено!)
    C++ — unsigned wrap-around 
    #include <iostream>
#include <limits>
#include <cstdint>

int main() {
    // Unsigned: wrap-around строго определён
    unsigned int u{std::numeric_limits<unsigned int>::max()};
    ++u;  // 4294967295 + 1 = 0  (mod 2^32)
    std::cout << "unsigned wrap: " << u << '\n';  // 0

    unsigned int z{0u};
    --z;  // 0 - 1 = UINT_MAX  (mod 2^32)
    std::cout << "unsigned underflow: " << z << '\n';  // 4294967295

    // uint8_t — ещё нагляднее (0..255)
    uint8_t byte{255u};
    ++byte;
    std::cout << "uint8_t wrap: " << (int)byte << '\n';  // 0

    return 0;
}

    Безопасные паттерны проверки переполнения

    Проверка перед операцией

    C++ — предварительная проверка 
    #include <iostream>
#include <limits>

// Безопасное сложение int: возвращает false при переполнении
bool safe_add(int a, int b, int& result) {
    if (b > 0 && a > std::numeric_limits<int>::max() - b) return false;
    if (b < 0 && a < std::numeric_limits<int>::min() - b) return false;
    result = a + b;
    return true;
}

int main() {
    int res{};
    int x{2'000'000'000};
    int y{2'000'000'000};

    if (safe_add(x, y, res))
        std::cout << "OK: " << res << '\n';
    else
        std::cout << "Переполнение! " << x << " + " << y << '\n';

    return 0;
}

    Intrinsic-функции GCC/Clang

    GCC и Clang предоставляют встроенные функции для атомарной проверки переполнения. Они генерируют эффективный машинный код, используя флаги процессора:

    C++ — __builtin_*_overflow (GCC/Clang) 
    #include <iostream>

int main() {
    int x{2'000'000'000};
    int y{2'000'000'000};
    int result{};

    // __builtin_add_overflow: вернёт true при переполнении
    if (__builtin_add_overflow(x, y, &result)) {
        std::cout << "Overflow detected!\n";
    } else {
        std::cout << "result = " << result << '\n';
    }

    long long a{9'000'000'000LL};
    long long b{9'000'000'000LL};
    long long c{};
    if (__builtin_add_overflow(a, b, &c))
        std::cout << "long long overflow!\n";
    else
        std::cout << "long long ok: " << c << '\n';

    return 0;
}

    Расширение через более широкий тип

    C++ — расширение через long long 
    #include <iostream>
#include <limits>

int safe_mul(int a, int b) {
    long long result{(long long)a * b};  // вычисляем в long long
    if (result < std::numeric_limits<int>::min() ||
        result > std::numeric_limits<int>::max()) {
        std::cout << "Overflow in mul!\n";
        return 0; // или throw
    }
    return (int)result;
}

int main() {
    std::cout << safe_mul(100'000, 100'000) << '\n';  // overflow
    std::cout << safe_mul(1'000, 1'000) << '\n';      // 1000000 — OK
    return 0;
}

    Флаги компилятора для обнаружения

    Флаги для поиска переполнений
    • -fsanitize=signed-integer-overflow — runtime-детектор для signed UB
    • -fsanitize=unsigned-integer-overflow — для unsigned wrap-around
    • -ftrapv — вызывает SIGABRT при signed overflow (debug)
    • -Wconversion, -Wsign-conversion — предупреждения при компиляции

    Используйте UBSan (-fsanitize=undefined) в тестах — он поймает большинство проблем с переполнением.
