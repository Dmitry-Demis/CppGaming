    Параметры и аргументы функций

    Функции становятся по-настоящему полезными, когда могут принимать данные извне. Для этого используются параметры и аргументы.

    Параметры vs аргументы

    • Параметр — переменная в определении функции. Это «слот», который будет заполнен при вызове.
    • Аргумент — конкретное значение, переданное при вызове. Оно копируется в параметр.

    Передача по значению

    По умолчанию аргументы передаются по значению — создаётся копия. Изменения параметра внутри функции не влияют на оригинал.

    C++ — передача по значению 
    // загрузка...

    Передача по ссылке

    Чтобы функция могла изменить переданную переменную, используется передача по ссылке — параметр объявляется с &. Тогда параметр — это псевдоним оригинала, а не копия.

    C++ — передача по ссылке 
    // загрузка...

    Передача по const-ссылке

    Для больших объектов (строки, векторы, структуры) копирование дорого. Передача по const& позволяет избежать копии, но запрещает изменение.

    C++ — const-ссылка 
    // загрузка...
    Способ Синтаксис Копия? Изменяет оригинал? Когда использовать
    По значению int x Да Нет Примитивы, маленькие типы
    По ссылке int& x Нет Да Нужно изменить аргумент
    По const-ссылке const int& x Нет Нет Большие объекты только для чтения

    Значения по умолчанию

    Параметрам можно задать значения по умолчанию. Если аргумент не передан — используется значение по умолчанию. Параметры с умолчаниями должны идти после параметров без умолчаний.

    C++ — значения по умолчанию

    Перегрузка функций

    В C++ можно иметь несколько функций с одним именем, но разными параметрами — это называется перегрузка (overloading). Компилятор выбирает нужную версию по типам аргументов.

    C++ — перегрузка функций
