    Forward declarations и ODR

    Компилятор читает файл сверху вниз. Если функция вызывается до её определения — нужно заранее сообщить компилятору о её существовании. Для этого используется forward declaration.

    Проблема порядка

    Без объявления вызов функции до её определения — ошибка компиляции. Forward declaration решает это: указываем заголовок функции (без тела) перед первым использованием.

    C++ — forward declaration 
    // загрузка...

    One Definition Rule (ODR)

    ODR — одно из фундаментальных правил C++: каждая функция или переменная может быть определена ровно один раз. Объявлений (declarations) может быть сколько угодно — они не создают код.

    C++ — ODR 
    // загрузка...
    Объявление vs Определение
    • Объявление — сообщает компилятору об имени и типе. Не генерирует код. Можно повторять.
    • Определение — содержит тело функции или инициализатор переменной. Генерирует код. Только одно на программу.
    • Каждое определение является объявлением, но не наоборот.

    Зачем это важно

    В реальных проектах функции разбиваются по нескольким файлам. Forward declarations (обычно в .h заголовочных файлах) позволяют одному файлу использовать функции из другого. Подробнее — в Главе 3.
