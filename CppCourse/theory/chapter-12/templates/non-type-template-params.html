    Нетиповые параметры шаблона

    Параметры шаблона бывают не только типами. Нетиповой параметр шаблона (non-type template parameter) — это константное значение, передаваемое в шаблон. Вы уже видели это в std::bitset<8> — число 8 и есть нетиповой параметр.

    Синтаксис

    C++ — нетиповой параметр int N
    #include <iostream>

template <int N> // N — нетиповой параметр типа int
void printN()
{
    std::cout << N << '\n';
}

int main()
{
    printN<5>();   // N = 5 → выводит 5
    printN<42>();  // N = 42 → выводит 42
    printN<'c'>(); // 'c' конвертируется в int → выводит 99
    return 0;
}

    Зачем это нужно: static_assert на этапе компиляции

    Параметры функций не могут быть constexpr, поэтому нельзя использовать static_assert с ними. Нетиповые параметры шаблона решают эту проблему:

    C++ — проверка на этапе компиляции
    #include <cmath>
#include <iostream>

// Обычная функция: ошибка только в runtime
double sqrtRuntime(double d)
{
    if (d < 0) throw std::invalid_argument("negative");
    return std::sqrt(d);
}

// Шаблон: ошибка на этапе компиляции!
template <double D> // C++20: float-point non-type params
double sqrtCompile()
{
    static_assert(D >= 0.0, "D must be non-negative");
    return std::sqrt(D);
}

int main()
{
    std::cout << sqrtCompile<9.0>()  << '\n'; // 3
    // sqrtCompile<-4.0>(); // compile error: static_assert failed!
    return 0;
}

    Практический пример: фиксированный размер массива

    C++ — шаблон с размером массива
    #include <iostream>
#include <array>

// Функция, которая принимает std::array любого размера
template <typename T, std::size_t N>
void printArray(const std::array<T, N>& arr)
{
    for (const auto& x : arr)
        std::cout << x << ' ';
    std::cout << '\n';
}

int main()
{
    std::array<int, 5> a { 1, 2, 3, 4, 5 };
    std::array<double, 3> b { 1.1, 2.2, 3.3 };

    printArray(a); // T=int, N=5
    printArray(b); // T=double, N=3
    return 0;
}
