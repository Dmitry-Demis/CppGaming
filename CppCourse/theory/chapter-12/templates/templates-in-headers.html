    Шаблоны функций

    Шаблоны в нескольких файлах

    Обычные функции можно объявить в заголовке и определить в .cpp-файле. С шаблонами так не работает — и это важно понять, чтобы не получать загадочные ошибки компоновщика.

    Почему шаблон нельзя разделить на .h и .cpp

    C++ — НЕ РАБОТАЕТ: шаблон в .cpp
    // add.h — только объявление
template <typename T>
T addOne(T x);

// add.cpp — определение в отдельном файле
template <typename T>
T addOne(T x) { return x + 1; }

// main.cpp
#include "add.h"
int main()
{
    addOne(1);   // linker error: unresolved external symbol
    addOne(2.3); // linker error: unresolved external symbol
    return 0;
}
// Компилятор видит объявление, но не может инстанциировать —
// определение в другом файле, который компилируется отдельно!

    Решение: определение в заголовочном файле

    C++ — правильно: шаблон в .h
    // add.h — определение прямо в заголовке
#ifndef ADD_H
#define ADD_H

template <typename T>
T addOne(T x)
{
    return x + 1;
}

#endif

// main.cpp
#include "add.h"
#include <iostream>

int main()
{
    std::cout << addOne(1)   << '\n'; // 2
    std::cout << addOne(2.3) << '\n'; // 3.3
    return 0;
}

    Почему это не нарушает ODR

    Шаблоны освобождены от части правила ODR: одинаковые определения в разных файлах допустимы. Инстанциированные функции неявно inline — компоновщик оставляет одну копию.

    Лучшая практика

    Шаблоны функций, нужные в нескольких файлах, определяйте в заголовочном файле (.h) и подключайте через #include.
