    Ссылки

    Возврат по ссылке

    Функция может возвращать ссылку вместо копии. Это эффективно для больших объектов, но требует осторожности: возвращаемый объект должен пережить функцию.

    Безопасный возврат по ссылке

    C++ — возврат ссылки на статическую переменную
    #include <iostream>
#include <string>

const std::string& getProgramName()
{
    static const std::string name { "Мой калькулятор" }; // static: живёт до конца программы
    return name; // безопасно: name не уничтожается при выходе из функции
}

int main()
{
    std::cout << getProgramName() << '\n'; // Мой калькулятор
    return 0;
}

    Опасность: висячая ссылка

    C++ — НИКОГДА не возвращайте ссылку на локальную переменную
    const std::string& getBad()
{
    const std::string local { "опасно" }; // локальная переменная
    return local; // ОШИБКА: local уничтожается при выходе из функции!
}                 // Возвращённая ссылка — висячая (dangling reference)

// Использование такой ссылки — неопределённое поведение!

    Безопасно: возврат параметра-ссылки

    C++ — возврат параметра по ссылке
    #include <iostream>
#include <string>

// Возвращаем ту строку, которая идёт первой по алфавиту
const std::string& firstAlphabetical(const std::string& a, const std::string& b)
{
    return (a < b) ? a : b; // безопасно: a и b живут в вызывающем коде
}

int main()
{
    std::string s1 { "Яблоко" };
    std::string s2 { "Апельсин" };
    std::cout << firstAlphabetical(s1, s2) << '\n'; // Апельсин
    return 0;
}

    In/out параметры

    Иногда функция получает объект, изменяет его и «возвращает» через тот же параметр. Такие параметры называются in/out параметрами.

    C++ — in/out параметр
    #include <cmath>
#include <iostream>

// sinOut и cosOut — out-параметры: функция записывает в них результат
void getSinCos(double degrees, double& sinOut, double& cosOut)
{
    constexpr double pi { 3.14159265358979 };
    double radians { degrees * pi / 180.0 };
    sinOut = std::sin(radians);
    cosOut = std::cos(radians);
}

int main()
{
    double s {}, c {};
    getSinCos(45.0, s, c);
    std::cout << "sin(45°) = " << s << '\n'; // ~0.707
    std::cout << "cos(45°) = " << c << '\n'; // ~0.707
    return 0;
}
    Лучшая практика

    Никогда не возвращайте ссылку на локальную (нестатическую) переменную или временный объект. Предпочитайте возврат по значению, если объект не слишком дорог для копирования.
