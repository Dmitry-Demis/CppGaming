    Ссылки

    Ссылка — это псевдоним для существующего объекта. Любая операция над ссылкой применяется к объекту, на который она ссылается. Ссылка — не копия, не указатель — это буквально другое имя для того же объекта.

    Синтаксис

    C++ — объявление ссылки
    #include <iostream>

int main()
{
    int x { 5 };
    int& ref { x }; // ref — ссылка на x (псевдоним)

    std::cout << x   << '\n'; // 5
    std::cout << ref << '\n'; // 5 (то же самое)

    ref = 7; // изменяем x через ссылку
    std::cout << x   << '\n'; // 7
    std::cout << ref << '\n'; // 7

    x = 10;
    std::cout << ref << '\n'; // 10 — ссылка всегда отражает x

    return 0;
}

    Правила ссылок

    📋 Ключевые правила
    • Ссылка должна быть инициализирована при объявлении
    • Ссылку нельзя «перепривязать» к другому объекту
    • Неконстантная ссылка может привязываться только к изменяемому lvalue
    • Ссылки не являются объектами — у них нет собственного адреса
    C++ — ссылку нельзя перепривязать
    int x { 5 }, y { 6 };
int& ref { x }; // ref привязана к x

ref = y; // НЕ перепривязывает ref к y!
         // Это присваивает значение y переменной x
std::cout << x << '\n'; // 6 (x изменился, не ref)

    Ссылки на const

    const int& может привязываться к чему угодно: изменяемым lvalue, константам и даже rvalue. При привязке к rvalue создаётся временный объект, время жизни которого продлевается до времени жизни ссылки.

    C++ — const-ссылки
    int x { 5 };
const int& r1 { x };  // OK: привязка к изменяемому lvalue

const int y { 10 };
const int& r2 { y };  // OK: привязка к const lvalue

const int& r3 { 42 }; // OK: привязка к rvalue (временный объект)
std::cout << r3 << '\n'; // 42 — временный объект жив пока жива r3

// int& bad { 42 }; // ОШИБКА: неконстантная ссылка не может привязаться к rvalue
    Лучшая практика

    Предпочитайте const-ссылки обычным, если не нужно изменять объект через ссылку.
