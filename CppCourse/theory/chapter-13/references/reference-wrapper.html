    Ссылки

    std::reference_wrapper

    Ссылки в C++ — не объекты. Их нельзя хранить в массивах или контейнерах. Но иногда нужно именно это: коллекция ссылок на существующие объекты. Для этого существует std::reference_wrapper.

    Проблема: массив ссылок невозможен

    C++ — почему нельзя 
    #include <array>

int main()
{
    int x { 1 }, y { 2 };

    // ОШИБКА: нельзя создать массив ссылок
    // std::array<int&, 2> refs { x, y };

    // Это работает, но создаёт массив ЗНАЧЕНИЙ, не ссылок:
    std::array values { x, y }; // std::array<int, 2> — копии!

    return 0;
}

    Решение: std::reference_wrapper

    std::reference_wrapper<T> из заголовка <functional> — это объект, который ведёт себя как изменяемая ссылка на T. Его можно хранить в массивах и контейнерах.

    C++ — массив "ссылок" 
    #include <array>
#include <functional> // std::reference_wrapper
#include <iostream>

int main()
{
    int x { 1 }, y { 2 }, z { 3 };

    // Массив "ссылок" на int
    std::array<std::reference_wrapper<int>, 3> refs { x, y, z };

    // Изменяем через .get()
    refs[1].get() = 99;

    std::cout << y << '\n'; // 99 — y изменился!

    // При выводе неявно конвертируется в int&
    for (int val : refs)
        std::cout << val << ' '; // 1 99 3

    return 0;
}

    std::ref и std::cref

    Вспомогательные функции std::ref() и std::cref() создают reference_wrapper и const reference_wrapper соответственно. Они короче и удобнее в использовании с auto.

    C++ — std::ref и std::cref 
    #include <functional>
#include <iostream>

void increment(std::reference_wrapper<int> r)
{
    r.get()++;
}

int main()
{
    int value { 10 };

    auto ref  { std::ref(value) };  // reference_wrapper<int>
    auto cref { std::cref(value) }; // reference_wrapper<const int>

    increment(ref);
    std::cout << value << '\n'; // 11

    // cref защищает от изменений:
    // increment(cref); // ОШИБКА компиляции

    return 0;
}
    Когда использовать

    std::reference_wrapper нужен когда требуется хранить ссылки в контейнерах (std::vector, std::array) или передавать ссылки в алгоритмы. В большинстве других случаев обычные ссылки предпочтительнее.
