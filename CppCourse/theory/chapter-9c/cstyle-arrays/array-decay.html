    C-style массивы

    Деградация массива (array decay)

    Одна из самых коварных особенностей C-style массивов — деградация (decay). В большинстве контекстов массив автоматически преобразуется в указатель на свой первый элемент, теряя при этом информацию о длине.

    Что такое деградация

    C++ — деградация в действии 
    #include <iostream>

int main()
{
    int arr[5] { 9, 7, 5, 3, 1 };

    // arr имеет тип int[5]
    // При использовании в выражении — деградирует в int*
    auto ptr { arr }; // ptr имеет тип int*, не int[5]!

    std::cout << (ptr == &arr[0]) << '\n'; // true: ptr указывает на первый элемент

    // sizeof показывает разницу:
    std::cout << sizeof(arr) << '\n';  // 20 (5 * 4 байта)
    std::cout << sizeof(ptr) << '\n';  // 8 (размер указателя)

    return 0;
}

    Передача в функцию: массив деградирует

    Когда C-style массив передаётся в функцию, он деградирует в указатель. Функция не знает длину массива — это главная опасность.

    C++ — передача массива 
    #include <iostream>
#include <iterator> // std::size

void printArray(const int arr[], int length) // arr — это int*, не int[]!
{
    // std::size(arr) здесь НЕ работает — arr уже деградировал
    for (int i { 0 }; i < length; ++i)
        std::cout << arr[i] << ' ';
    std::cout << '\n';
}

int main()
{
    const int primes[] { 2, 3, 5, 7, 11 };

    // Нужно явно передавать длину:
    printArray(primes, static_cast<int>(std::size(primes)));

    return 0;
}

    Опасности деградации

    C++ — типичные ошибки 
    #include <iostream>

void badSizeof(int arr[])
{
    // ОШИБКА: sizeof(arr) = размер указателя, не массива!
    std::cout << sizeof(arr) / sizeof(arr[0]) << '\n'; // неверный результат
}

int main()
{
    int arr[] { 1, 2, 3, 4, 5 };

    // В main: sizeof работает правильно
    std::cout << sizeof(arr) / sizeof(arr[0]) << '\n'; // 5 — верно

    // В функции: sizeof работает неправильно
    badSizeof(arr); // выведет 2 или 1 (размер указателя / 4)

    return 0;
}
    Почему C-style массивы устарели

    Деградация — главная причина, по которой C-style массивы считаются опасными в современном C++. Используй std::array или std::vector вместо них. Они не деградируют и всегда знают свою длину.

    Когда деградации нет

    Массив не деградирует в нескольких случаях:

    • При использовании в sizeof() или typeid()
    • При взятии адреса через &arr
    • При передаче по ссылке: const int (&arr)[5]
    C++ — передача по ссылке (без деградации) 
    #include <iostream>
#include <iterator>

// Принимает массив по ссылке — деградации нет, но только для int[5]
void printArray(const int (&arr)[5])
{
    std::cout << std::size(arr) << '\n'; // 5 — работает!
    for (int x : arr)
        std::cout << x << ' ';
    std::cout << '\n';
}

// Шаблонная версия — для любой длины
template <std::size_t N>
void printAny(const int (&arr)[N])
{
    std::cout << N << '\n'; // N — длина массива
}

int main()
{
    int arr[] { 1, 2, 3, 4, 5 };
    printArray(arr);
    printAny(arr);
    return 0;
}
