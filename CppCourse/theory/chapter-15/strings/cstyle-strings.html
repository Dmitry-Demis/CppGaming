    C-строки

    C-строка — это просто массив символов (char[]), заканчивающийся специальным символом null-терминатором ('\0'). Это наследие языка C, которое до сих пор встречается в старом коде.

    Null-терминатор: зачем он нужен

    Когда C-строка деградирует в указатель, информация о длине теряется. Null-терминатор позволяет функциям (например, std::cout) знать, где строка заканчивается.

    C++ — C-строки 
    #include <cstring> // strlen
#include <iostream>

int main()
{
    // Инициализация строковым литералом — компилятор добавляет '\0'
    const char str[] { "hello" }; // char[6]: 'h','e','l','l','o','\0'

    // Вывод: std::cout читает до '\0'
    std::cout << str << '\n'; // hello

    // Длина строки (без '\0'):
    std::cout << std::strlen(str) << '\n'; // 5

    // Изменение отдельных символов:
    char mutable_str[] { "hello" };
    mutable_str[0] = 'H';
    std::cout << mutable_str << '\n'; // Hello

    // Нельзя присвоить новую строку целиком:
    // mutable_str = "world"; // ОШИБКА компиляции

    return 0;
}

    Символьные константы: const char*

    C++ — const char* vs constexpr string_view 
    #include <iostream>
#include <string_view>

int main()
{
    // Старый способ: const char*
    const char* name1 { "Alex" }; // указатель на строковый литерал

    // Современный способ: constexpr string_view (предпочтительно)
    constexpr std::string_view name2 { "Alex" };

    std::cout << name1 << '\n'; // Alex
    std::cout << name2 << '\n'; // Alex

    // Опасность с const char*:
    // std::cout << &name1[0]; // выведет адрес как строку — OK
    // char c { 'Q' };
    // std::cout << &c; // ОПАСНО: нет null-терминатора!

    return 0;
}
    Используй std::string и std::string_view

    В современном C++ C-строки (char[], const char*) следует избегать. Используй std::string для изменяемых строк и constexpr std::string_view для строковых констант.

    Когда C-строки всё ещё встречаются

    • В старом коде (legacy codebase)
    • При работе с C API (например, printf, системные вызовы)
    • Строковые литералы в коде — это всегда const char[]
    • Глобальные constexpr массивы символов
