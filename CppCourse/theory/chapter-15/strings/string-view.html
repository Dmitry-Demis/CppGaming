    Строки

    std::string_view

    std::string_view (C++17) — это лёгкий «вид» на строку. Он не владеет данными и не копирует их — только хранит указатель и длину. Это делает его очень эффективным для передачи строк в функции.

    Что такое string_view

    C++ — string_view
    #include <string>
#include <string_view>
#include <iostream>

// Принимает любой тип строки без копирования
void printLength(std::string_view sv)
{
    std::cout << sv << " (длина: " << sv.length() << ")\n";
}

int main()
{
    std::string s { "Hello, world!" };
    const char* cstr { "C-style string" };

    printLength(s);      // std::string → string_view (без копии)
    printLength(cstr);   // const char* → string_view (без копии)
    printLength("literal"); // литерал → string_view (без копии)

    // Суффикс sv для string_view литерала:
    using namespace std::string_view_literals;
    constexpr std::string_view greeting { "Hello"sv };
    std::cout << greeting << '\n';

    return 0;
}

    string vs string_view

    Свойство std::string std::string_view
    Владеет данными ✅ Да ❌ Нет (только смотрит)
    Копирование Дорогое Дешёвое (2 слова)
    Изменение ✅ Можно ❌ Нельзя
    Принимает литералы Дорого (копирует) Дёшево (без копии)
    Когда использовать Нужно хранить/изменять Только читать, передавать

    Опасность: висячий string_view

    C++ — ОПАСНО: string_view на уничтоженную строку
    std::string_view getDangling()
{
    std::string local { "hello" }; // локальная строка
    return local; // ОПАСНО: local уничтожается, string_view висит!
}

// Правильно: string_view на строковый литерал (живёт всю программу)
std::string_view getSafe()
{
    return "hello"; // OK: литерал живёт всю программу
}
    Лучшая практика

    Используйте std::string_view для параметров функций, которые только читают строку. Не возвращайте string_view на локальную переменную. Для хранения строк используйте std::string.
