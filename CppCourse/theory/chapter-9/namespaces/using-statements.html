    using-объявления и using-директивы

    Многие учебники начинают с using namespace std;. Если вы видите это в современном коде — это тревожный сигнал. Разберёмся, почему.

    Квалифицированные и неквалифицированные имена

    Квалифицированное имя включает пространство имён: std::cout. Неквалифицированное — без него: cout.

    using-объявление

    Позволяет использовать конкретный идентификатор без квалификатора в текущей области видимости:

    C++ — using-объявление 
    #include <iostream>

int main()
{
    using std::cout; // только cout из std
    cout << "Hello world!\n"; // можно без std::
    return 0;
} // using-объявление заканчивается здесь

    using-директива

    Открывает доступ ко всем именам из пространства имён без квалификатора:

    C++ — using-директива (избегайте!) 
    #include <iostream>

int main()
{
    using namespace std; // ВСЕ имена из std доступны без квалификатора
    cout << "Hello world!\n";
    return 0;
}

    Почему using namespace std опасно

    Проблема 1: конфликт имён при обновлении библиотеки:

    C++ — скрытый конфликт 
    // Ваш код работает сегодня:
using namespace Foo;
print(); // вызывает вашу функцию ::print()

// После обновления Foo добавляет Foo::print() —
// теперь вызов становится неоднозначным или меняет поведение!

    Проблема 2: неочевидность происхождения имён:

    C++ — с директивой vs без 
    // Плохо: откуда foo()?
using namespace NS;
foo(); // это NS::foo() или пользовательская функция?

// Хорошо: сразу понятно
NS::foo(); // из библиотеки NS
foo();     // пользовательская функция

    Область действия using-операторов

    using-операторы подчиняются обычным правилам области видимости. Ограничьте их блоком, если нужно:

    C++ — ограничение области действия 
    int main()
{
    {
        using namespace Foo;
        // Foo:: доступно здесь
    } // using namespace Foo заканчивается

    {
        using namespace Goo;
        // Goo:: доступно здесь
    }
    return 0;
}
    Лучшая практика

    Предпочитайте явные квалификаторы (std::cout). Избегайте using-директив. using-объявления допустимы в .cpp файлах после всех #include. Никогда не используйте using-операторы в заголовочных файлах.
