    Безымянные и inline-пространства имён

    Два специальных вида пространств имён, которые решают конкретные задачи: безымянные — для изоляции кода в файле, inline — для версионирования API.

    Безымянные (анонимные) пространства имён

    Пространство имён без имени. Всё внутри него автоматически получает внутреннюю компоновку и доступно в родительском пространстве имён без квалификатора:

    C++ — безымянное пространство имён 
    #include <iostream>

namespace // безымянное
{
    void doSomething() // доступна только в этом файле
    {
        std::cout << "v1\n";
    }
}

int main()
{
    doSomething(); // вызываем без квалификатора
    return 0;
}

    Это эквивалентно объявлению всех функций как static, но удобнее для группы идентификаторов:

    C++ — эквивалент через static 
    static void doSomething() // то же самое, но только для одной функции
{
    std::cout << "v1\n";
}
    💡 Преимущество безымянных пространств имён

    В отличие от static, безымянные пространства имён также дают внутреннюю компоновку типам (классам, перечислениям), для которых нет аналога static.

    Inline-пространства имён

    Inline-пространство имён — инструмент для версионирования API. Содержимое inline-пространства доступно в родительском без квалификатора, но при этом компоновка остаётся внешней:

    C++ — inline namespace для версионирования 
    #include <iostream>

inline namespace V1 // текущая версия (inline = по умолчанию)
{
    void doSomething() { std::cout << "V1\n"; }
}

namespace V2 // новая версия
{
    void doSomething() { std::cout << "V2\n"; }
}

int main()
{
    V1::doSomething(); // явно V1
    V2::doSomething(); // явно V2
    doSomething();     // V1 (inline-версия)
    return 0;
}

    Когда готовы перейти на V2 — просто переставьте inline:

    C++ — переключение на новую версию 
    namespace V1 { void doSomething() { std::cout << "V1\n"; } }

inline namespace V2 // теперь V2 — версия по умолчанию
{
    void doSomething() { std::cout << "V2\n"; }
}

// doSomething() теперь вызывает V2
// Старый код может явно использовать V1::doSomething()

    Лучшая практика

    Используйте безымянные пространства имён для кода, который должен оставаться локальным для файла. Не используйте их в заголовочных файлах.
