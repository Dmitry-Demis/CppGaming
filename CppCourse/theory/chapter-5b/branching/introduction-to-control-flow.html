    Введение в управление потоком

    По умолчанию CPU выполняет инструкции строго сверху вниз. Конкретная последовательность инструкций, которую проходит программа, называется путём выполнения (execution path). Программа, у которой путь всегда одинаков, называется прямолинейной.

    C++ — прямолинейная программа 
    // загрузка...

    Прямолинейные программы ограничены: они не могут реагировать на ввод, повторять действия или обрабатывать ошибки. Для этого нужны операторы управления потоком. Когда такой оператор передаёт управление в другое место — это называется ветвлением (branching).

    Категории операторов управления потоком

    В C++ есть шесть категорий:

    Категория Что делает Операторы C++
    Условные Выполняют код только при выполнении условия if, else, switch
    Прыжки Передают управление в другое место goto, break, continue
    Вызовы функций Прыжок в функцию и возврат обратно вызов функции, return
    Циклы Повторяют блок кода ноль или более раз while, do-while, for, range-for
    Остановка Завершают программу std::exit(), std::abort()
    Исключения Специальный механизм обработки ошибок try, throw, catch
    C++ — ветвление и цикл 
    // загрузка...

    Почему это важно

    До этой главы программы могли делать только фиксированный набор действий. Управление потоком — особенно циклы — открывает возможность писать программы с реальной пользой: обрабатывать коллекции данных, валидировать ввод, реализовывать алгоритмы.
