    Оператор if и блоки

    Условный оператор — это инструкция, которая решает, выполнять ли связанный с ней код. Самый базовый условный оператор в C++ — if.

    Синтаксис if / else

    Базовая форма:

    Синтаксис
    if (условие)
    инструкция_если_истина;

if (условие)
    инструкция_если_истина;
else
    инструкция_если_ложь;

    Если условие истинно — выполняется первая инструкция. Если ложно и есть else — выполняется вторая. Ровно одна из двух веток выполняется всегда.

    Блоки для нескольких инструкций

    После if или else можно написать только одну инструкцию. Чтобы выполнить несколько — нужен блок { }. Блок считается одной составной инструкцией.

    Ловушка отступов

    Отступ не влияет на логику. Только { } объединяют инструкции в блок. Без блока вторая строка выполняется всегда, независимо от условия.

    Неявные блоки и область видимости

    Даже без явных { } компилятор неявно создаёт блок вокруг каждой ветки. Это значит, что переменная, объявленная внутри ветки if, уничтожается в конце этой ветки и недоступна снаружи.

    Почему это не компилируется
    if (true)
    int x{ 5 };   // x живёт только внутри неявного блока
else
    int x{ 6 };   // другой x, тоже уничтожается

std::cout << x;   // ошибка: x не существует здесь
    ? Лучшая практика

    Всегда используйте явные блоки { } после if и else, особенно пока учитесь. Это предотвращает случайные ошибки при добавлении новых строк и обеспечивает совместимость с if constexpr (C++23).

    if-else против if-if

    Ключевое различие: if-else выполняет код только после первого истинного условия. if-if проверяет все условия независимо.

    Когда else не нужен

    Если каждая ветка заканчивается return, else не добавляет смысла — после первого return функция уже завершилась. Многие разработчики предпочитают убирать лишние else для чистоты кода: 

    // Оба варианта эквивалентны:
if (a) return 'a';
else if (b) return 'b';  // else здесь лишний

if (a) return 'a';
if (b) return 'b';       // чище
