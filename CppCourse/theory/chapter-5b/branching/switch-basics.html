    Оператор switch — основы

    Когда нужно сравнить одно значение с несколькими конкретными вариантами, цепочка if-else if становится громоздкой. Оператор switch решает эту задачу компактнее и потенциально эффективнее.

    Синтаксис switch

    Синтаксис
    switch (выражение)
{
    case константа_1:
        // инструкции
        break;
    case константа_2:
        // инструкции
        break;
    default:
        // инструкции при отсутствии совпадения
        break;
}

    Компилятор вычисляет выражение и ищет совпадающую метку case. Если совпадение найдено — выполнение переходит к этой метке. Оператор break завершает блок switch и передаёт управление следующей инструкции после него.

    Почему switch может быть эффективнее if-else

    При достаточном числе вариантов компилятор реализует switch через таблицу переходов (jump table): массив адресов, индексированный значением выражения. Переход к нужной ветке выполняется за O(1) — одной операцией обращения к памяти, независимо от числа меток case. Цепочка if-else if проверяет условия последовательно, что даёт O(n) в худшем случае.

    Метка default и отсутствие совпадения

    Метка default выполняется, если ни одна метка case не совпала с выражением. Она необязательна.

    Нет совпадения и нет default

    Если значение выражения не совпадает ни с одним case и метка default отсутствует, весь блок switch пропускается целиком. Выполнение продолжается с первой инструкции после закрывающей скобки }.

    Ограничение типов

    Выражение в switch должно иметь целочисленный или перечислимый тип: int, char, long, enum и т.д. Вещественные числа (float, double) и строки (std::string) не поддерживаются — для них используйте if-else.

    Константы в метках case

    Значения после case должны быть целочисленными константными выражениями, известными на этапе компиляции. Переменные и вызовы функций здесь недопустимы.
