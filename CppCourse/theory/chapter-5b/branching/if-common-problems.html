    Типичные ошибки с if

    Оператор if прост в синтаксисе, но легко допустить ошибку, которая компилируется без предупреждений и даёт неверный результат. Разберём четыре самых распространённых случая.

    Dangling else — «висячий else»

    При вложенных if без блоков возникает неоднозначность: к какому if относится else? Правило C++: else всегда привязывается к ближайшему незакрытому if в том же блоке.

    Dangling else
    if (x >= 0)
    if (x <= 20)
        std::cout << "от 0 до 20\n";
    else                          // привязан к (x <= 20), не к (x >= 0)!
        std::cout << "отрицательное\n";  // BUG при x = 21

    Правило

    Чем меньше уровней вложенности — тем меньше вероятность ошибки. Если видите три и более уровня if подряд — это сигнал к рефакторингу.

    Null statement — пустой оператор

    Пустой оператор — это просто ; на отдельной строке. Он ничего не делает. Случайно поставить его после if — классическая ошибка: условный блок становится пустым, а следующая инструкция выполняется всегда.

    Лишняя точка с запятой
    if (isAdmin());          // <-- точка с запятой! if выполняет пустой оператор
    deleteDatabase();    // выполняется ВСЕГДА, не условно

    Присваивание вместо сравнения

    Yoda conditions

    Некоторые разработчики пишут константу слева: if (0 == x). Тогда случайное if (0 = x) — ошибка компиляции, а не тихий баг. Современные компиляторы с -Wall предупреждают о if (x = 0), поэтому «йода-условия» сейчас редкость.
