    Оператор goto

    goto — один из старейших операторов управления потоком. В современном C++ его почти не используют, но понимание его механики и ограничений важно для чтения устаревшего кода и понимания, почему он был вытеснен структурированными конструкциями.

    Синтаксис и метки

    goto передаёт управление на метку оператора (statement label) — идентификатор с двоеточием:

    Синтаксис
    goto имя_метки;   // переход

// ... код ...

имя_метки:        // метка — точка назначения
    инструкция;

    Метки имеют область видимости функции (function scope): они видны во всей функции, независимо от блоков { }. Переход возможен только в пределах одной функции — перейти в другую функцию нельзя.

    Ограничения

    Нельзя перепрыгивать через инициализацию

    Если goto перепрыгивает через инициализацию переменной, которая остаётся в области видимости в точке назначения, компилятор выдаёт ошибку. Переменная была бы в области видимости, но без инициализации.

    C++ — goto для выхода из вложенных циклов 
    // загрузка...

    Спагетти-код и почему goto избегают

    Произвольные переходы goto разрушают линейную структуру кода. Поток выполнения становится похож на запутанные нити — отсюда термин «спагетти-код». Такой код трудно читать, отлаживать и модифицировать: невозможно понять состояние программы в произвольной точке, не отслеживая все возможные пути переходов.

    Рекомендация

    Избегайте goto в новом коде. Для любой задачи, которую решает goto, существует структурированная альтернатива: циклы, функции, исключения или флаговые переменные. Единственное спорное исключение — выход из глубоко вложенных циклов, но и там предпочтительнее вынести циклы в отдельную функцию.
