    Завершение программы

    C++ предоставляет несколько функций для явного завершения программы. Понимание разницы между ними важно для корректной обработки ресурсов.

    std::exit()

    std::exit(код) завершает программу нормально. Перед завершением она:

    • вызывает функции, зарегистрированные через std::atexit;
    • сбрасывает буферы потоков ввода/вывода;
    • освобождает статические объекты.
    std::exit не уничтожает локальные переменные

    Деструкторы локальных переменных при вызове std::exit не вызываются. Если локальные объекты управляют ресурсами (файлы, сетевые соединения, мьютексы), они не будут корректно освобождены. Это одна из причин предпочитать исключения явному вызову std::exit.

    std::atexit()

    std::atexit(функция) регистрирует функцию очистки, которая будет вызвана при нормальном завершении программы. Можно зарегистрировать несколько функций — они вызываются в порядке, обратном регистрации (LIFO).

    std::abort() и std::terminate()

    Аварийное завершение
    Функция Поведение
    std::exit(n) Нормальное завершение, вызывает atexit-функции
    std::abort() Аварийное завершение, atexit-функции не вызываются, генерирует SIGABRT
    std::terminate() Вызывается при необработанном исключении, по умолчанию вызывает abort()

    Рекомендация

    Предпочитайте исключения

    Явные вызовы std::exit и std::abort обходят механизм деструкторов и нарушают принцип RAII. В большинстве случаев правильнее выбросить исключение: оно позволяет вызывающему коду обработать ошибку, а деструкторы корректно освободят ресурсы при раскрутке стека.
