    Fallthrough и область видимости в switch

    Одна из самых частых ловушек в switch — пропущенный break. Понимание механизма fallthrough и правил области видимости позволяет избежать трудноуловимых ошибок.

    Механизм fallthrough

    Когда выполнение достигает метки case, оно продолжается последовательно вниз до тех пор, пока не встретит break, return или конец блока switch. Это называется fallthrough («проваливание»).

    Опасность непреднамеренного fallthrough

    Если забыть break, выполнение перейдёт в следующий case и выполнит его код, даже если значение не совпадает. Большинство компиляторов выдают предупреждение об этом.

    // загрузка...

    Стекирование меток case

    Намеренное использование fallthrough — группировка нескольких значений для одного действия. Несколько меток case подряд без кода между ними называют стекированием меток:

    Группировка значений
    switch (grade)
{
    case 'A':
    case 'B':
    case 'C':
        std::cout << "Зачёт\n";  // выполняется для A, B и C
        break;
    default:
        std::cout << "Незачёт\n";
        break;
}

    Область видимости переменных в switch

    Весь блок switch образует одну область видимости. Переменная, объявленная в одном case, видна во всех последующих метках.

    Инициализация через метку — ошибка компиляции

    Нельзя инициализировать переменную в одном case, если другой case может «перепрыгнуть» через эту инициализацию. Компилятор запрещает это, так как переменная оказалась бы в области видимости, но без инициализации. 

    switch (x)
{
    case 1:
        int y {5};  // ошибка: инициализация перепрыгивается case 2
        break;
    case 2:
        std::cout << y;  // y в области видимости, но не инициализирована
        break;
}

    Решение — явный блок { } внутри case. Он создаёт отдельную область видимости, и переменная не «утекает» в другие метки:

    ? Правильный подход
    switch (x)
{
    case 1:
    {
        int y {5};  // y видна только внутри этого блока
        std::cout << y;
        break;
    }
    case 2:
        // y здесь недоступна
        break;
}
