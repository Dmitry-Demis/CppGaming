    Преобразования типов

    Арифметические преобразования

    Многие бинарные операторы требуют, чтобы оба операнда были одного типа. Когда типы разные, компилятор применяет обычные арифметические преобразования (usual arithmetic conversions), чтобы привести их к общему типу.

    Какие операторы используют эти правила

    📋 Операторы с арифметическими преобразованиями
    • Бинарная арифметика: + - * / %
    • Сравнение: < > <= >= == !=
    • Побитовые: & ^ |
    • Тернарный оператор ?: (кроме условия)

    Иерархия типов

    Правило простое: операнд с более низким рангом приводится к типу операнда с более высоким.

    Примеры

    C++ — арифметические преобразования 
    #include <iostream>
#include <typeinfo>

int main()
{
    // int + double → double (int повышается до double)
    std::cout << typeid(2 + 3.5).name() << '\n'; // double, результат 5.5

    // short + short → int (оба повышаются до int)
    short a { 4 }, b { 5 };
    std::cout << typeid(a + b).name() << '\n'; // int, результат 9

    // Классическая ошибка: целочисленное деление
    double d = 10 / 4;   // 10 и 4 — оба int → int деление → 2.0
    double e = 10.0 / 4; // 10.0 — double → 4 повышается → 2.5

    std::cout << d << '\n'; // 2
    std::cout << e << '\n'; // 2.5

    return 0;
}

    Ловушка: знаковые и беззнаковые в выражениях

    C++ — неожиданный результат 
    #include <iostream>

int main()
{
    // 5u — unsigned int, 10 — int
    // int приводится к unsigned int → -5 становится огромным числом
    std::cout << 5u - 10 << '\n'; // 4294967291, а не -5!

    // -3 приводится к unsigned → больше 5u → false!
    std::cout << std::boolalpha << (-3 < 5u) << '\n'; // false (!)

    return 0;
}
    Лучшая практика

    Не смешивайте знаковые и беззнаковые типы в арифметических выражениях. Компилятор обычно не предупреждает об этом по умолчанию.
