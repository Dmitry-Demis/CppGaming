    Преобразования типов

    Сужающие преобразования и brace-инициализация

    Сужающее преобразование (narrowing conversion) — это потенциально опасное числовое преобразование, при котором целевой тип может не вместить все значения исходного. Это подмножество числовых преобразований, которое C++ выделяет особо.

    Что считается сужающим

    📋 Сужающие преобразования
    • Вещественный тип → целый тип
    • Вещественный тип → более узкий вещественный (если значение не constexpr)
    • Целый тип → вещественный (если значение не constexpr или не помещается точно)
    • Целый тип → другой целый, который не может представить все значения исходного

    Почему brace-инициализация запрещает сужение

    Это одна из главных причин предпочитать { } вместо = или ( ):

    C++ — brace-init защищает от сужения 
    int a = 3.5;   // OK: неявное сужение (предупреждение)
int b(3.5);    // OK: неявное сужение (предупреждение)
int c { 3.5 }; // ОШИБКА КОМПИЛЯЦИИ: brace-init запрещает сужение

double d = 3.5;
int e { d };   // ОШИБКА: d не constexpr, сужение запрещено

    Явное сужение через static_cast

    Если сужение необходимо — сделайте его явным. Это документирует намерение и убирает предупреждения:

    C++ — явное сужение 
    void someFcn(int i) {}

double d { 5.0 };
someFcn(d);                      // плохо: неявное сужение
someFcn(static_cast<int>(d));   // хорошо: явное, намеренное

int i { static_cast<int>(d) };  // хорошо: явное сужение в brace-init

    Исключение для constexpr

    Если значение известно на этапе компиляции (constexpr) и помещается в целевой тип — преобразование не считается сужающим:

    C++ — constexpr-исключение 
    constexpr int n1 { 5 };
unsigned int u1 { n1 };  // OK: 5 помещается в unsigned int

constexpr int n2 { -5 };
unsigned int u2 { n2 };  // ОШИБКА: -5 не помещается в unsigned int

// Практическое применение: не нужны суффиксы литералов
unsigned int u { 5 };    // OK (не нужно писать 5u)
float f { 1.5 };         // OK (не нужно писать 1.5f)
short s { 5 };           // OK (нет суффикса для short)
    Лучшая практика

    Избегайте сужающих преобразований. Если они необходимы — используйте static_cast, чтобы сделать их явными.
