    Преобразования типов

    Числовые преобразования

    Числовые преобразования — это все преобразования между числовыми типами, которые не являются повышениями. В отличие от повышений, они не всегда безопасны.

    Пять видов числовых преобразований

    C++ — виды числовых преобразований 
    // 1. Целое → другое целое (не повышение)
short s = 3;       // int → short
long l = 3;        // int → long
char ch = s;       // short → char

// 2. Вещественное → другое вещественное (не повышение)
float f = 3.0;     // double → float
long double ld = 3.0; // double → long double

// 3. Вещественное → целое
int i = 3.5;       // double → int (дробная часть теряется!)

// 4. Целое → вещественное
double d = 3;      // int → double

// 5. Целое или вещественное → bool
bool b1 = 3;       // int → bool (0 = false, всё остальное = true)
bool b2 = 3.0;     // double → bool

    Примеры потери данных

    C++ — потеря данных при преобразовании 
    #include <iostream>
#include <iomanip>

int main()
{
    // double → int: дробная часть отбрасывается (не округляется!)
    int i = 3.9;
    std::cout << i << '\n'; // 3, а не 4!

    // double → float: потеря точности
    float f = 0.123456789;
    std::cout << std::setprecision(9) << f << '\n'; // 0.123456791

    // int → char: переполнение
    int big { 30000 };
    char c = big; // char: диапазон -128..127
    std::cout << static_cast<int>(c) << '\n'; // 48 (неожиданно!)

    return 0;
}

    Ловушка: знаковые и беззнаковые

    Смешивание знаковых и беззнаковых типов — частый источник ошибок. Компилятор обычно не предупреждает об этом по умолчанию:

    C++ — неожиданный результат со знаковыми/беззнаковыми 
    #include <iostream>

int main()
{
    // int -5 преобразуется в unsigned int → огромное число
    unsigned int u { static_cast<unsigned int>(-5) };
    std::cout << u << '\n'; // 4294967291

    // Сравнение: -3 < 5u → false! (потому что -3 → огромное unsigned)
    std::cout << std::boolalpha << (-3 < 5u) << '\n'; // false (!)

    return 0;
}
    Лучшая практика

    Избегайте смешивания знаковых и беззнаковых типов в арифметических выражениях. Это одна из главных причин избегать unsigned int в обычном коде.
