    Преобразования типов

    Неявные преобразования типов

    Преобразование типа — это процесс конвертации данных из одного типа в другой. Когда компилятор делает это автоматически, без явного указания программиста, это называется неявным преобразованием (implicit conversion, или coercion).

    Зачем нужны преобразования

    Разные типы хранят одно и то же значение по-разному на уровне битов. Например, целое число 3 и вещественное 3.0 — это разные битовые паттерны:

    Поэтому нельзя просто скопировать биты — нужно именно преобразовать значение. Компилятор создаёт временный объект целевого типа с правильным значением.

    Когда происходит неявное преобразование

    C++ — случаи неявного преобразования 
    // 1. Инициализация переменной значением другого типа
double d { 3 };     // int → double

// 2. Возвращаемое значение другого типа
float foo() { return 3.0; } // double → float

// 3. Бинарный оператор с разными типами
double div { 4.0 / 3 }; // int 3 → double

// 4. Не-bool в условии
if (5) { }  // int → bool

// 5. Аргумент функции другого типа
void bar(long l) {}
bar(3);  // int → long

    Когда преобразование невозможно

    Если компилятор не знает, как преобразовать тип — ошибка компиляции:

    C++ — ошибка: нет преобразования 
    int x { "14" }; // ошибка: нет преобразования из const char* в int

int y { 3.5 };  // ошибка при brace-init: сужающее преобразование запрещено
    💡 Ключевой момент

    Преобразование не меняет исходные данные — оно создаёт новый временный объект целевого типа. Исходная переменная остаётся нетронутой.
