    Явные преобразования и static_cast

    Когда компилятор не может или не должен делать преобразование автоматически, программист запрашивает его явно — через оператор приведения (cast). В C++ есть 5 видов приведений, но в повседневном коде нужен в основном один.

    Виды приведений

    Оператор Назначение Безопасен?
    static_cast Преобразование между связанными типами (compile-time) ✅ Да
    dynamic_cast Приведение в иерархии наследования (runtime) ✅ Да
    const_cast Добавление/снятие const ⚠️ Только добавление
    reinterpret_cast Переинтерпретация битов ❌ Нет
    C-style (type)x Комбинация нескольких C++ приведений ❌ Нет

    static_cast — основной инструмент

    static_cast<TargetType>(expression) — явное, безопасное, проверяемое на этапе компиляции.

    C++ — static_cast в действии 
    #include <iostream>

int main()
{
    // Проблема: целочисленное деление
    int x { 10 }, y { 4 };
    double d1 = x / y;                        // 2.0 (целочисленное деление!)
    double d2 = static_cast<double>(x) / y;  // 2.5 (правильно)

    // char → int: вывести числовой код символа
    char c { 'a' };
    std::cout << static_cast<int>(c) << '\n'; // 97

    // Явное сужение: документируем намерение
    double pi { 3.14159 };
    int approx { static_cast<int>(pi) }; // 3

    std::cout << d1 << ' ' << d2 << '\n'; // 2 2.5
    return 0;
}

    Почему C-style cast опасен

    C++ — C-style cast vs static_cast 
    int x { 10 }, y { 4 };

// C-style: работает, но непрозрачен — может выполнить reinterpret_cast!
std::cout << (double)x / y << '\n';

// static_cast: явно, безопасно, легко найти в коде
std::cout << static_cast<double>(x) / y << '\n';
    💡 Живое применение

    Типичные случаи для static_cast:

    • Вещественное деление из целых переменных: static_cast<double>(a) / b
    • Вывод числового кода символа: static_cast<int>(ch)
    • Явное сужение при вызове функции: someFcn(static_cast<int>(d))
    • Преобразование size_t → int при работе с контейнерами
    Лучшая практика

    Используйте static_cast для явных преобразований. Избегайте C-style cast. Не используйте const_cast и reinterpret_cast без веской причины.
