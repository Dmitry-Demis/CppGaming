    Ввод и вывод: iostream

    Библиотека <iostream> предоставляет инструменты для взаимодействия с пользователем через консоль. iostream = input/output stream. Чтобы использовать её возможности, нужно подключить заголовок в начале файла: 

    #include <iostream>

    std::cout — вывод

    std::cout (character output) выводит данные в стандартный поток вывода (консоль). Оператор << передаёт данные в поток — можно представить его как конвейер, который несёт данные в сторону std::cout. Можно цеплять несколько значений подряд.

    C++ — цепочка вывода 
    // загрузка...

    Перевод строки: \n vs endl

    Отдельные вызовы std::cout не добавляют перенос строки автоматически. Чтобы перейти на новую строку, нужно явно вывести символ переноса. Есть два способа: std::endl и \n.

    std::endl делает две вещи: переводит курсор на новую строку и принудительно сбрасывает буфер вывода. Сброс буфера — медленная операция, поэтому при частом использовании std::endl производительность заметно падает.

    \n — просто символ новой строки, без сброса буфера. Его можно встроить прямо в строку или вывести отдельно. В большинстве случаев именно он предпочтителен.

    Формы записи \n
    • "текст\n" — встроен в строку (наиболее распространённо)
    • "\n" — отдельный строковый литерал (тоже допустимо)
    • '\n' — символьный литерал (предпочтительно, когда \n стоит отдельно)
    '\n' vs "\n" — в чём разница

    В C++ одинарные кавычки обозначают один символ (char), двойные — строку (const char[]). '\n' — это ровно один байт. "\n" — это массив из двух байт: символ переноса и нулевой терминатор.

    Для вывода через std::cout разницы нет — оба работают одинаково. Но семантически точнее использовать '\n', когда выводите одиночный символ, а не строку. Это же правило распространяется на любые одиночные символы: ' ', 'a' и т.д.

    Обратный слэш, не прямой

    \n пишется через обратный слэш (\). Прямой слэш /n — это просто два символа, никакого переноса строки не будет. Пробел внутри кавычек (' \n') тоже даст неожиданный результат.

    Буферизация вывода

    Вывод через std::cout не попадает на экран мгновенно. Данные сначала накапливаются в буфере — области памяти, которая периодически сбрасывается (flush) на устройство вывода. Это значительно ускоряет работу: одна операция записи на экран дешевле, чем десятки отдельных.

    Буфер и аварийное завершение

    Если программа упала или была остановлена до сброса буфера, данные из него не будут выведены на экран. Именно поэтому при отладке иногда кажется, что вывод "пропал".

    std::cin — ввод

    std::cin (character input) читает данные из стандартного потока ввода (клавиатура). Оператор >> извлекает данные из потока и записывает в переменную — конвейер идёт в обратную сторону: от std::cin к переменной. Можно читать несколько значений подряд.

    Инициализируйте переменную перед вводом

    Переменную перед вводом стоит инициализировать, даже если значение сразу перезапишется. Это защищает от неопределённого поведения, если извлечение вдруг не удастся — например, пользователь введёт букву вместо числа.

    Как передать данные в std::cin в редакторе

    Нажмите на шестерёнку ⚙ в правом верхнем углу редактора и введите нужные данные в поле Stdin — они попадут в std::cin при запуске программы.

    C++ — cout и cin 
    // загрузка...
    Не нужен \n после ввода

    При чтении через std::cin не нужно выводить '\n' после приглашения — пользователь сам нажмёт Enter, и курсор перейдёт на новую строку.

    cin и пробелы

    std::cin >> x пропускает ведущие пробелы и читает до следующего пробела/переноса. Для чтения строки с пробелами используйте std::getline(std::cin, str).

    Буферизация ввода

    Ввод тоже буферизован. Символы, которые вы набираете, накапливаются во входном буфере. Нажатие Enter фиксирует строку — она целиком попадает в буфер вместе с символом '\n'. Оператор >> затем извлекает данные из начала буфера по одному значению за раз. Буфер работает по принципу FIFO: первым пришёл — первым ушёл.

    Это значит, что если ввести 4 5 и нажать Enter, первый >> извлечёт 4, второй — 5, не ожидая нового ввода. Остаток буфера доступен для следующих извлечений.

    C++ — два числа из одного ввода 
    // загрузка...
    Как работает оператор >>
    1. Если std::cin в состоянии ошибки — извлечение немедленно прерывается.
    2. Ведущие пробелы, табы и переносы строк отбрасываются.
    3. Если буфер пуст — оператор ждёт ввода от пользователя.
    4. Символы читаются подряд, пока не встретится пробел, перенос или неподходящий символ.
    5. Прочитанные символы конвертируются в нужный тип и присваиваются переменной.
    6. Непрочитанные символы остаются в буфере для следующего извлечения.

    Если ни одного подходящего символа не нашлось (например, ввели букву вместо числа), переменная получает значение 0, а std::cin переходит в состояние ошибки — все последующие извлечения будут немедленно завершаться неудачей, пока поток не будет сброшен.

    Зачем std::

    std:: — это пространство имён (namespace) стандартной библиотеки. Оно предотвращает конфликты имён: std::cout и std::cin живут в пространстве std, а не в глобальном. Можно написать using namespace std;, но это считается плохой практикой.

    Почему using namespace std; — плохая практика

    Директива using namespace std; импортирует все имена из стандартной библиотеки в глобальное пространство имён. Это создаёт несколько проблем:

    • Конфликты имён. Если в вашем коде есть функция или переменная с тем же именем, что и в std (например, count, find, sort), компилятор не сможет однозначно определить, что именно вы имеете в виду.
    • Скрытые зависимости. Читая код, невозможно сразу понять, откуда пришло то или иное имя — из стандартной библиотеки или из вашего проекта.
    • Проблемы при масштабировании. В небольшом файле конфликт маловероятен, но в крупном проекте с десятками заголовков риск резко возрастает.

    Пишите std::cout, std::cin и т.д. явно — это делает код понятнее и надёжнее.

    Мнемоника: << и >>

    Новички часто путают операторы и объекты. Простое правило:

    • std::cout и std::cin всегда стоят слева от оператора.
    • << используется с std::cout — стрелка показывает направление данных к консоли: std::cout << 4
    • >> используется с std::cin — стрелка показывает направление данных от клавиатуры к переменной: std::cin >> x
