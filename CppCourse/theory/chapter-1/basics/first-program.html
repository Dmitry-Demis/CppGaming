    Первая программа

    Пришло время применить всё, что мы изучили, и написать полноценную программу. Разберём процесс разработки от задачи до рабочего кода.

    Процесс разработки

    Хороший программист не сразу пишет код. Сначала нужно понять задачу, потом спроектировать решение, и только потом реализовать его.

    Шаги разработки
    1. Понять задачу — что программа должна делать?
    2. Спроектировать решение — какие данные нужны, какие шаги?
    3. Написать код — реализовать шаги
    4. Скомпилировать — исправить синтаксические ошибки
    5. Протестировать — проверить на разных входных данных
    6. Улучшить — рефакторинг, оптимизация

    Пример: сумма двух чисел

    Задача: написать программу, которая запрашивает два числа у пользователя и выводит их сумму.

    Итеративное улучшение

    Не пытайтесь написать идеальный код с первого раза. Начните с простого рабочего решения, затем улучшайте его.

    Советы начинающему
    • Компилируйте часто — не пишите 100 строк без проверки
    • Тестируйте на граничных значениях: 0, отрицательные числа
    • Читайте сообщения об ошибках — компилятор указывает на строку
    • Используйте осмысленные имена переменных
    • Добавляйте комментарии к неочевидным местам
    Частая ошибка новичков

    Попытка написать всю программу сразу, а потом исправить все ошибки. Лучше: написал 5–10 строк > скомпилировал > исправил > продолжил.
