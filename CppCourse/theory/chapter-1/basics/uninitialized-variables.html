    Неинициализированные переменные и UB

    Одна из самых коварных ошибок в C++ — использование переменной до того, как ей присвоено значение.

    Что такое неинициализированная переменная

    Когда вы объявляете переменную без начального значения (int x;), компилятор резервирует память, но не очищает её. В этой памяти может быть что угодно — «мусор» от предыдущих операций.

    C++ — неинициализированные переменные 
    // загрузка...

    Undefined Behavior (UB)

    Неопределённое поведение (UB) — ситуация, когда стандарт C++ не определяет, что должна делать программа. Компилятор может делать что угодно:

    ? Что может произойти при UB
    • Программа выведет случайное число
    • Программа упадёт с ошибкой сегментации
    • Программа будет работать «правильно» сегодня, но сломается завтра
    • Компилятор оптимизирует код непредсказуемым образом
    UB — это не предупреждение, это катастрофа

    UB не всегда вызывает видимую ошибку. Программа может «работать» годами, а потом сломаться после обновления компилятора или смены платформы. Всегда инициализируйте переменные.

    Правило: всегда инициализировать

    Современный C++ рекомендует инициализировать переменные при объявлении. Если значение ещё неизвестно — используйте 0 или {}.

    ? Хорошая практика
    • int x{}; — инициализация нулём
    • int x = 0; — явный ноль
    • Объявляйте переменную как можно ближе к месту первого использования
