    Структура программы

    Прежде чем писать код, нужно понять, из чего состоит программа на C++. Каждая программа — это набор инструкций (statements), которые компилятор превращает в машинный код.

    Инструкции

    Инструкция — это наименьшая самостоятельная единица вычисления. Большинство инструкций заканчиваются точкой с запятой ;. Программа выполняет инструкции последовательно, сверху вниз.

    Типы инструкций
    • Инструкции объявления — создают переменные: int x { 5 };
    • Инструкции выражений — вычисляют значение: x = x + 1;
    • Инструкции вывода — выводят данные: std::cout << x;
    • Инструкции возврата — завершают функцию: return 0;

    Блоки кода

    Несколько инструкций можно объединить в блок кода, заключив их в фигурные скобки { }. Блок воспринимается компилятором как единая составная инструкция.

    Блок кода сам по себе — редкость. Чаще всего блоки используются как тело функции, условия или цикла:

    Функция — это именованный блок кода. Тело функции тоже заключается в фигурные скобки.

    Функция main()

    Каждая программа на C++ должна содержать функцию main() — это точка входа, с которой начинается выполнение. Когда программа запускается, ОС вызывает main().

    C++ — функция main 
    // загрузка...

    Возвращаемое значение main()

    main() возвращает int — код завершения для операционной системы. return 0 означает «программа завершилась успешно». Любое ненулевое значение сигнализирует об ошибке.

    Параметры main()

    Функция main() может принимать аргументы командной строки:

    argc (argument count) — количество переданных аргументов.
    argv (argument vector) — массив строк, где argv[0] — имя программы.

    Для простых программ используется вариант без параметров: int main().

    C++ — функция main 
    // загрузка...

    Препроцессорные директивы

    Строки, начинающиеся с #, — это директивы препроцессора. Они обрабатываются до компиляции. Самая частая — #include, которая вставляет содержимое другого файла.

    #include <iostream>

    Подключает стандартную библиотеку ввода/вывода. Без неё std::cout и std::cin недоступны. Угловые скобки <> означают системный заголовочный файл.

    Этапы компиляции

    Превращение исходного кода в исполняемый файл проходит три этапа:

    1️⃣
    Препроцессор — обрабатывает директивы #include, #define и другие. Вставляет содержимое заголовочных файлов, раскрывает макросы.
    2️⃣
    Компилятор — переводит каждый .cpp-файл в объектный файл (.o / .obj) с машинным кодом. Проверяет синтаксис и типы.
    3️⃣
    Компоновщик (линкер) — объединяет все объектные файлы и библиотеки в единый исполняемый файл (.exe / ELF).
