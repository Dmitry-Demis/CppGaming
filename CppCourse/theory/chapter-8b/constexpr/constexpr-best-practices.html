    Продвинутые constexpr

    constexpr-функции: локальные переменные и лучшие практики

    Разберём, что можно делать внутри constexpr-функций: использовать изменяемые локальные переменные, вызывать другие constexpr-функции с параметрами, и когда вообще стоит помечать функцию как constexpr.

    Локальные переменные внутри constexpr

    Внутри constexpr и consteval-функций можно объявлять и изменять локальные переменные, которые сами не являются constexpr. При вычислении при компиляции компилятор «выполняет» функцию и вычисляет результат.

    C++
    #include <iostream>

consteval int doSomething(int x, int y)
{
    x = x + 2;        // можно изменять параметры
    int z { x + y };  // можно объявлять не-constexpr локальные переменные
    if (x > y)
        z = z - 1;    // и изменять их
    return z;
}

int main()
{
    constexpr int g { doSomething(5, 6) }; // 5+2=7, z=7+6=13, 7>6 → z=12
    std::cout << g << '\n'; // 12
    return 0;
}

    Параметры как аргументы вложенных constexpr-вызовов

    Хотя параметры constexpr-функции не являются константными выражениями, их можно передавать в другие constexpr-функции внутри. Если внешняя функция вычисляется при компиляции, компилятор знает значения всех параметров и может вычислить вложенные вызовы тоже при компиляции.

    C++
    #include <iostream>

constexpr int goo(int c) { return c; }

constexpr int foo(int b) // b — не constexpr
{
    return goo(b); // OK: если foo вычисляется при компиляции, goo(b) тоже
}

int main()
{
    constexpr int r { foo(5) }; // вычисляется при компиляции
    std::cout << r << '\n';
    return 0;
}

    Вызов не-constexpr функций из constexpr

    constexpr-функция может вызывать обычные функции, но только в runtime-контексте. При compile-time вычислении такой вызов — ошибка.

    C++
    #include <iostream>
#include <type_traits>

int runtimeOnly(int x) { return x * 3; }
constexpr int compileOnly(int x) { return x * 2; }

constexpr int smart(int x)
{
    if (std::is_constant_evaluated())
        return compileOnly(x); // только при компиляции
    else
        return runtimeOnly(x); // только при выполнении
}

int main()
{
    constexpr int a { smart(5) }; // compile-time: 10
    int b { smart(5) };           // runtime: 15
    std::cout << a << ' ' << b << '\n';
    return 0;
}

    Когда делать функцию constexpr?

    📋 Правило
    • Если функция может быть частью константного выражения — сделайте её constexpr
    • Чистые функции (одинаковый результат для одинаковых аргументов, нет побочных эффектов) — почти всегда стоит делать constexpr
    • Если функция не может вычисляться при компиляции (ввод/вывод, глобальное состояние) — не помечайте constexpr
    Почему constexpr даже без compile-time использования?

    Даже если сейчас функция вызывается только с runtime-аргументами, constexpr не мешает и может помочь в будущем. Добавить constexpr позже — значит рисковать сломать что-то и перетестировать. Лучше сразу.

    Лучшая практика

    Если функция может быть частью константного выражения — сделайте её constexpr, даже если сейчас она используется только в runtime-контексте.
