    consteval и принудительное вычисление при компиляции

    constexpr-функция может вычисляться при компиляции, но не обязана. C++20 вводит consteval — ключевое слово для функций, которые обязаны вычисляться только при компиляции. Такие функции называются immediate functions.

    consteval

    C++20
    #include <iostream>

consteval int greater(int x, int y)
{
    return x > y ? x : y;
}

int main()
{
    constexpr int g { greater(5, 6) };   // OK: при компиляции
    std::cout << greater(5, 6) << '\n'; // OK: при компиляции

    int x { 5 };
    // std::cout << greater(x, 6);       // ошибка: x не константа
    return 0;
}

    constexpr vs consteval

    📋 Сравнение
    • constexpr — может вычисляться и при компиляции, и при выполнении
    • consteval — только при компиляции; вызов с runtime-аргументами — ошибка
    • Параметры consteval-функции не являются constexpr (для единообразия с constexpr)

    Принудительное compile-time вычисление constexpr

    Чтобы гарантировать вычисление constexpr-функции при компиляции, можно передать её результат в consteval-функцию:

    C++20
    #include <iostream>

// Вспомогательная consteval-функция для принудительного compile-time вычисления
consteval auto CONSTEVAL(auto value) { return value; }

constexpr int greater(int x, int y) { return x > y ? x : y; }

int main()
{
    std::cout << greater(5, 6) << '\n';          // может быть при выполнении
    std::cout << CONSTEVAL(greater(5, 6)) << '\n'; // гарантировано при компиляции
    return 0;
}

    Когда использовать consteval?

    Лучшая практика

    Используйте consteval, если функция должна вычисляться исключительно при компиляции (например, генерация таблиц, проверки на этапе компиляции). В остальных случаях предпочитайте constexpr.
