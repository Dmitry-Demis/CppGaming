    Продвинутые constexpr

    Принудительное compile-time вычисление

    constexpr-функция не гарантирует вычисление при компиляции везде. Иногда нужно заставить компилятор вычислить результат именно при компиляции. C++20 предлагает для этого consteval.

    Способ 1: constexpr-переменная

    Самый простой способ — сохранить результат в constexpr-переменную. Это вынуждает компилятор вычислить функцию при компиляции.

    C++
    #include <iostream>

constexpr int greater(int x, int y) { return x > y ? x : y; }

int main()
{
    constexpr int g { greater(5, 6) }; // гарантировано при компиляции
    std::cout << g << '\n';
    return 0;
}

    Недостаток: приходится вводить лишнюю переменную. C++20 решает это через consteval.

    Способ 2: consteval-обёртка (C++20)

    Аргументы consteval-функции всегда вычисляются в константном контексте. Если передать в неё вызов constexpr-функции, тот тоже будет вычислен при компиляции.

    C++20
    #include <iostream>

// Обёртка: принимает любое значение и возвращает его, гарантируя compile-time вычисление аргумента
consteval auto CONSTEVAL(auto value) { return value; }

constexpr int greater(int x, int y) { return x > y ? x : y; }

int main()
{
    std::cout << greater(5, 6) << '\n';          // может быть при выполнении
    std::cout << CONSTEVAL(greater(5, 6)) << '\n'; // гарантировано при компиляции
    return 0;
}
    Как это работает

    Поскольку CONSTEVAL — это consteval-функция, её аргументы вычисляются в константном контексте. Значит, greater(5, 6) внутри тоже обязан вычислиться при компиляции.

    std::is_constant_evaluated (C++20)

    std::is_constant_evaluated() возвращает true, если функция выполняется в обязательном константном контексте. Это позволяет писать разное поведение для compile-time и runtime:

    C++20
    #include <iostream>
#include <type_traits>

constexpr int compute(int x)
{
    if (std::is_constant_evaluated())
        return x * x;   // compile-time ветка
    else
        return x + x;   // runtime ветка
}

int main()
{
    constexpr int a { compute(5) }; // compile-time: 25
    int b { compute(5) };           // runtime: 10 (или 25 — зависит от оптимизации)
    std::cout << a << ' ' << b << '\n';
    return 0;
}
    Важно

    std::is_constant_evaluated() возвращает true только когда компилятор обязан вычислить выражение при компиляции, а не просто когда он решил это сделать по оптимизации. В C++23 появился if consteval с более чистым синтаксисом и теми же правилами.
