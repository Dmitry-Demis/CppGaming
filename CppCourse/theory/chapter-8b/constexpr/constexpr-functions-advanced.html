    Продвинутые constexpr

    constexpr-функции: детали

    В главе «Константы» мы познакомились с constexpr-функциями. Здесь разберём тонкости: когда вычисление при компиляции гарантировано, а когда нет, почему параметры не являются constexpr, и зачем функции неявно inline.

    Не всегда при компиляции

    constexpr-функция может вычисляться при компиляции, но не обязана. Компилятор вычисляет её при компиляции только тогда, когда это требуется — то есть результат используется там, где нужно константное выражение.

    C++
    #include <iostream>

constexpr int getValue(int x) { return x; }

int main()
{
    constexpr int a { getValue(5) }; // гарантировано при компиляции
    int b { getValue(5) };           // может быть при выполнении — компилятор решает
    std::cout << a << ' ' << b << '\n';
    return 0;
}
    📋 Когда гарантировано compile-time?
    • Всегда — результат используется в контексте, требующем константного выражения (инициализатор constexpr, размер массива, аргумент шаблона)
    • Вероятно — все аргументы — константные выражения, но результат не обязан быть константным
    • Скорее при выполнении — хотя бы один аргумент не является константным выражением, но его значение известно при компиляции
    • Всегда при выполнении — аргумент зависит от ввода пользователя или других runtime-данных
    Ключевая идея

    Compile-time вычисление constexpr-функций гарантировано только там, где требуется константное выражение.

    Диагностика: ошибки только при compile-time вызове

    Компилятор не обязан проверять, может ли constexpr-функция вычисляться при компиляции, пока она не вызвана в таком контексте. Функция может успешно компилироваться для runtime, но падать при compile-time вызове.

    C++ — ошибка только при constexpr
    #include <iostream>

int getValue(int x) { return x; } // обычная функция, не constexpr

constexpr int foo(int x)
{
    return getValue(x); // вызов не-constexpr функции
}

int main()
{
    int a { foo(5) };           // OK: вычисляется при выполнении
    // constexpr int b { foo(5) }; // ошибка: foo не может вычислиться при компиляции
    std::cout << a << '\n';
    return 0;
}
    Лучшая практика

    Всегда проверяйте constexpr-функцию в контексте, требующем константного выражения — например, инициализируя constexpr-переменную. Иначе ошибка проявится позже.

    Параметры не являются constexpr

    Параметры constexpr-функции не являются constexpr и не могут быть объявлены таковыми. Это сделано намеренно: функция может вызываться и с runtime-аргументами, поэтому параметры нельзя использовать там, где требуется константное выражение.

    C++ — параметры не constexpr
    constexpr int foo(int b) // b — не constexpr
{
    // constexpr int b2 { b }; // ошибка: b не константное выражение
    return b * 2;
}

int main()
{
    constexpr int a { 5 };
    constexpr int r { foo(a) }; // OK: a — константное выражение, передаётся как аргумент
    return 0;
}
    Нужны параметры-константы?

    Если нужны параметры, которые являются константными выражениями, используйте нетиповые параметры шаблонов (template <int N>).

    constexpr-функции неявно inline

    Для вычисления при компиляции компилятор должен видеть полное определение функции, а не только объявление. Чтобы не нарушать ODR при включении в несколько единиц трансляции, constexpr-функции неявно inline.

    Поэтому constexpr-функции, используемые в нескольких .cpp-файлах, обычно определяют в заголовочных файлах.

    Правило

    Компилятор должен видеть полное определение constexpr-функции до её вызова в контексте, требующем константного выражения. Предварительного объявления недостаточно.
