    Mersenne Twister и распределения

    Mersenne Twister — самый популярный PRNG в C++. Он даёт хорошее качество и приемлемую производительность для большинства задач. В стандартной библиотеке доступен через <random>.

    Базовое использование

    C++ — std::mt19937 
    #include <iostream>
#include <random>

int main()
{
    std::mt19937 mt {}; // 32-битный Mersenne Twister

    for (int i { 1 }; i <= 5; ++i)
        std::cout << mt() << '\n'; // генерируем случайное число

    return 0;
}

    Проблема: при каждом запуске программы числа одинаковые, потому что seed не задан.

    Правильная инициализация seed

    Два способа получить разный seed при каждом запуске:

    C++ — seed через системные часы 
    #include <random>
#include <chrono>

std::mt19937 mt {
    static_cast<std::mt19937::result_type>(
        std::chrono::steady_clock::now().time_since_epoch().count()
    )
};
    C++ — seed через std::random_device (рекомендуется) 
    #include <random>

std::mt19937 mt { std::random_device{}() };

    Улучшенная инициализация через seed_seq

    Mersenne Twister имеет состояние 19937 бит, а один seed — только 32 бита. Это называется недоинициализацией. Лучше использовать std::seed_seq с несколькими случайными значениями:

    C++ — качественная инициализация 
    #include <random>

std::random_device rd {};
std::seed_seq ss { rd(), rd(), rd(), rd(), rd(), rd(), rd(), rd() }; // 8 случайных значений
std::mt19937 mt { ss }; // хорошо инициализированный генератор

    Распределения: числа в нужном диапазоне

    mt() возвращает числа от 0 до ~4 млрд. Чтобы получить число в нужном диапазоне, используют распределения:

    C++ — бросок кубика d6 
    #include <iostream>
#include <random>

int main()
{
    std::random_device rd {};
    std::seed_seq ss { rd(), rd(), rd(), rd(), rd(), rd(), rd(), rd() };
    std::mt19937 mt { ss };

    // Равномерное распределение от 1 до 6
    std::uniform_int_distribution die6 { 1, 6 };

    for (int i { 1 }; i <= 10; ++i)
        std::cout << die6(mt) << ' ';
    std::cout << '\n';

    return 0;
}

    Типичные ошибки

    Не создавайте генератор внутри функции при каждом вызове
    // Плохо: генератор пересоздаётся и переинициализируется при каждом вызове
int getCard()
{
    std::mt19937 mt { std::random_device{}() }; // ошибка!
    return std::uniform_int_distribution{ 1, 52 }(mt);
}
    💡 Правило

    Создайте генератор один раз и используйте его повторно. Инициализируйте seed только один раз.

    Отладка программ с рандомом

    При отладке удобно зафиксировать seed, чтобы программа воспроизводила одно и то же поведение:

    C++ — фиксированный seed для отладки 
    std::mt19937 mt { 42 }; // фиксированный seed — всегда одна последовательность
// После отладки замените на нормальную инициализацию
