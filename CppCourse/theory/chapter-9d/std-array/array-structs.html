    std::array

    std::array со структурами и перечислениями

    Элементами std::array могут быть не только числа, но и любые объекты: структуры, классы, перечисления. Однако инициализация таких массивов имеет свои тонкости.

    Массив структур

    C++ — массив структур 
    #include <array>
#include <iostream>
#include <string_view>

struct Student {
    int id {};
    std::string_view name {};
};

int main()
{
    // Явно указываем тип каждого элемента — одинарные скобки
    constexpr std::array students {
        Student{ 1, "Алиса" },
        Student{ 2, "Борис" },
        Student{ 3, "Вера" }
    };

    for (const auto& s : students)
        std::cout << s.id << ": " << s.name << '\n';

    return 0;
}

    Двойные скобки: почему они нужны

    Внутри std::array хранится C-style массив. Когда мы не указываем тип элемента явно при каждом инициализаторе, компилятор путается: он думает, что первый { } инициализирует внутренний C-style массив целиком. Нужна дополнительная пара скобок.

    C++ — двойные скобки 
    #include <array>
#include <iostream>

struct House {
    int number {};
    int floors {};
    int rooms {};
};

int main()
{
    // Без явного типа — нужны двойные скобки {{ }}
    constexpr std::array<House, 3> houses {{ // <-- двойные скобки
        { 13, 1, 7 },
        { 14, 2, 5 },
        { 15, 2, 4 }
    }};

    for (const auto& h : houses)
        std::cout << "Дом " << h.number
                  << ": " << h.floors * h.rooms << " комнат\n";

    return 0;
}
    Brace elision — когда можно обойтись одними скобками

    Если тип каждого элемента указан явно (Student{...}), или элементы — скалярные значения, одинарных скобок достаточно. Двойные скобки нужны только когда тип не указан явно и элемент сам является агрегатом.

    std::array с перечислениями

    Удобный паттерн: использовать перечисление как индекс массива. Это делает код самодокументируемым и позволяет проверять корректность через static_assert.

    C++ — enum как индекс 
    #include <array>
#include <iostream>
#include <string_view>

namespace Color {
    enum Type { black, red, blue, max_colors };

    using namespace std::string_view_literals;
    constexpr std::array names { "чёрный"sv, "красный"sv, "синий"sv };
    static_assert(std::size(names) == max_colors); // проверка на этапе компиляции
}

int main()
{
    // Индексируем по перечислению
    std::cout << Color::names[Color::red] << '\n'; // красный

    // Перебор всех цветов
    for (std::size_t i { 0 }; i < Color::max_colors; ++i)
        std::cout << Color::names[i] << '\n';

    return 0;
}

    static_assert здесь — страховка: если добавить новый цвет в перечисление, но забыть добавить строку в массив, программа не скомпилируется.
