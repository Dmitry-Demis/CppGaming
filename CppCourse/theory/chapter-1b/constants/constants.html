    Константы и constexpr

    Константные переменные

    Константа — это значение, которое не может измениться после инициализации. Константы делают программу надёжнее: компилятор не даст случайно изменить значение, которое должно оставаться неизменным.

    Ключевое слово const

    C++ — константные переменные
    #include <iostream>

int main()
{
    const double gravity { 9.8 };  // константа: нельзя изменить
    // gravity = 10.0;             // ОШИБКА: const нельзя присвоить

    const int sidesInSquare { 4 }; // именованная константа

    // Константу можно инициализировать из переменной:
    int age { 25 };
    const int constAge { age }; // OK: инициализатор может быть не-const
    // constAge = 26;           // ОШИБКА

    std::cout << gravity << '\n';       // 9.8
    std::cout << sidesInSquare << '\n'; // 4
    return 0;
}

    Зачем использовать константы

    Константы vs макросы

    В старом коде можно встретить #define PI 3.14159. Это плохая практика: макросы не имеют типа, не подчиняются правилам области видимости и могут вызвать неожиданные ошибки.

    C++ — константа vs макрос
    // Плохо: макрос
#define MAX_SIZE 100  // нет типа, нет области видимости, нет отладки

// Хорошо: константная переменная
constexpr int maxSize { 100 }; // тип int, область видимости, видна отладчику

    constexpr — константа времени компиляции

    constexpr гарантирует, что значение вычисляется на этапе компиляции. Это быстрее и позволяет использовать значение там, где требуется константное выражение (например, размер массива).

    C++ — constexpr
    #include <iostream>

int main()
{
    constexpr double pi { 3.14159265358979 };
    constexpr int maxStudents { 30 };

    // constexpr можно использовать как размер массива:
    int scores[maxStudents] {};

    // Вычисление тоже может быть constexpr:
    constexpr double twoPi { 2.0 * pi }; // вычисляется при компиляции

    std::cout << "2π = " << twoPi << '\n'; // 6.28318...
    return 0;
}
    💡 const vs constexpr
    • const — значение не изменится, но может быть вычислено в runtime
    • constexpr — значение вычисляется при компиляции, гарантированно
    • Предпочитайте constexpr для числовых констант
    Лучшая практика

    Делайте переменные константными везде, где это возможно. Используйте constexpr для числовых констант. Избегайте «магических чисел» — давайте константам говорящие имена.
