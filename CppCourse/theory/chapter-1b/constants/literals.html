    Литералы и суффиксы

    Литерал — это значение, вписанное прямо в код: 5, 3.14, 'a', "hello". Каждый литерал имеет тип, который компилятор определяет автоматически.

    Типы литералов по умолчанию

    Литерал Примеры Тип по умолчанию
    Целое число 5, 0, -3 int
    Булево true, false bool
    Вещественное 1.2, 3.14 double (не float!)
    Символ 'a', '\n' char
    Строка "hello" const char[N]

    Суффиксы для изменения типа

    C++ — суффиксы литералов
    #include <iostream>

int main()
{
    // Целочисленные суффиксы
    auto a { 5 };    // int
    auto b { 5u };   // unsigned int
    auto c { 5L };   // long
    auto d { 5LL };  // long long
    auto e { 5uLL }; // unsigned long long

    // Вещественные суффиксы
    auto f { 3.14 };  // double (по умолчанию)
    auto g { 3.14f }; // float (нужен суффикс f!)
    auto h { 3.14L }; // long double

    // Типичная ошибка: предупреждение компилятора
    float bad { 4.1 };  // 4.1 — double, конвертируется в float с потерей точности
    float good { 4.1f }; // правильно: float-литерал

    return 0;
}

    Научная нотация

    C++ — научная нотация
    double avogadro { 6.02e23 };   // 6.02 × 10²³
double proton   { 1.6e-19 };   // 1.6 × 10⁻¹⁹
double pi       { 3.14159 };   // обычная нотация

    Магические числа — антипаттерн

    Магическое число — литерал без объяснения смысла. Такой код трудно читать и поддерживать.

    C++ — магические числа vs именованные константы
    // Плохо: что такое 30? Почему 30?
int total { numClassrooms * 30 };
setMax(30);

// Хорошо: смысл очевиден
constexpr int studentsPerClass { 30 };
int total { numClassrooms * studentsPerClass };
setMax(studentsPerClass);

// Если нужно изменить — меняем в одном месте!
